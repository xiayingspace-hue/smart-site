## 需求描述

### 背景与目标

Site Engineer 在施工现场需要第一时间知道图纸的局部改动，并能方便地在 APP 端查阅详情、确认已读，确保施工依据始终是最新内容。

### 用户故事

```
作为 Site Engineer
我想要 在 APP 端收到图纸局部更新的推送通知
以便 及时了解与我相关的图纸改动，不遗漏任何施工依据的变更
```

```
作为 Site Engineer
我想要 在 APP 端查看某张图纸的所有局部更新内容
以便 全面掌握该图纸在当前版本下的所有补充说明和改动记录
```

## 功能需求

### 1. 推送通知接收

#### 1.1 通知来源

当管理员在 PC 端发布局部更新后，系统自动向**已被分配该图纸**的 Site Engineer 推送 App Push 通知。

#### 1.2 通知格式

| 字段 | 内容 |
|------|------|
| 标题 | Drawing Update — {drawingCode} |
| 正文 | {markupTitle}. Please review the latest markup for {drawingCode} {drawingName}. |
| 角标 | 新通知数累计显示在 APP 图标角标 |
| 跳转 | 点击通知跳转到对应图纸的**局部更新 Tab** |

---

### 2. 图纸详情页调整

#### 2.1 新增 Markups Tab

在 REQ-003 图纸详情页（或图纸列表项点击展开的详情面板）新增 **Markups** Tab：

```
┌──────────────────────────────────────┐
│  ARCH-001  首层平面图         V3     │
│  Architectural  |  🟢 Active         │
├──────────────────────────────────────┤
│  [Info]  [Markups (2)]               │  ← Tab 切换
├──────────────────────────────────────┤
│  ...                                 │
```

- Tab 标题显示该图纸当前 `ACTIVE` 局部更新数量，0 时仅显示 "Markups"
- 点击通知推送跳转时，自动激活 Markups Tab

#### 2.2 Markups Tab 内容

```
┌──────────────────────────────────────┐
│  ┌────────────────────────────────┐  │
│  │  [New]  A轴节点详图修正        │  │  ← 未读标记
│  │  Apr 8, 2026  ·  张三          │  │
│  │  A-C轴 / 3-5层                 │  │
│  │  A轴与3轴交叉节点详图已更新，  │  │
│  │  新增钢筋排布说明...           │  │
│  │  📎 node-detail.pdf            │  │
│  │           [✓ Mark as Read]     │  │
│  └────────────────────────────────┘  │
│  ┌────────────────────────────────┐  │
│  │  C区消防管道路由修正            │  │
│  │  Apr 7, 2026  ·  张三          │  │
│  │  C区 / B1层                    │  │
│  │  消防主管道路由变更，详见附图   │  │
│  │  📎 route-update.png           │  │
│  │           ✓ Read               │  │  ← 已确认状态
│  └────────────────────────────────┘  │
└──────────────────────────────────────┘
```

---

### 3. 局部更新详情

#### 3.1 触发方式

点击 Markups Tab 中的局部更新卡片，进入局部更新详情页。

#### 3.2 详情页内容

```
┌──────────────────────────────────────┐
│  ←  A轴节点详图修正                  │
├──────────────────────────────────────┤
│  ARCH-001 首层平面图  V3             │
│  Published by 张三  ·  Apr 8, 2026  │
├──────────────────────────────────────┤
│                                      │
│  Affected Area                       │
│  A-C轴 / 3-5层                       │
│                                      │
│  Description                         │
│  A轴与3轴交叉节点详图已更新，新增     │
│  钢筋排布说明，请对照施工。           │
│                                      │
│  Attachments                         │
│  ┌────────────────────────────────┐  │
│  │  📄 node-detail.pdf   512 KB  │  │
│  │              [Preview / Open] │  │
│  └────────────────────────────────┘  │
│                                      │
├──────────────────────────────────────┤
│         [✓ Mark as Read]             │  ← 未确认时显示
│         ✓ You've confirmed reading   │  ← 已确认后替换
└──────────────────────────────────────┘
```

#### 3.3 [Mark as Read] 交互

- 点击 [Mark as Read] 调用 `/drawing/markup/confirm`
- 提交中 loading，防止重复点击
- 成功后按钮替换为 "✓ You've confirmed reading"，不可再次点击
- 已确认状态持久化，重新进入页面仍显示已确认

---

### 4. 未读标记与角标

| 场景 | 处理 |
|------|------|
| 有新局部更新未确认 | 图纸列表该图纸条目右上角显示橙色小圆点 |
| 局部更新卡片未确认 | 卡片左上角显示 `[New]` 橙色标签 |
| 点击通知后进入详情 | 对应卡片的 `[New]` 标签在**确认后**消失（不自动消失） |
| 全部确认后 | 图纸条目角标消失 |

## 验收标准

### 推送通知
- [ ] 局部更新发布后，已被分配该图纸的 Site Engineer 收到 App Push 通知
- [ ] 未被分配该图纸的 Site Engineer 不收到推送
- [ ] 点击通知正确跳转到对应图纸的 Markups Tab

### Markups Tab
- [ ] Tab 标题显示当前 ACTIVE 局部更新数量
- [ ] 列表仅显示 `ACTIVE` 状态的局部更新，已汇总（Merged）的不显示
- [ ] 局部更新按发布时间降序排列，最新在上

### 局部更新详情
- [ ] 详情页正确展示标题、说明、影响区域、发布人、发布时间
- [ ] 附件可点击预览或打开
- [ ] 未确认时显示 [Mark as Read] 按钮
- [ ] 确认成功后按钮变为 "✓ You've confirmed reading"，不可再次点击
- [ ] 刷新或重新进入页面确认状态仍正确保留

### 未读标记
- [ ] 有未确认局部更新的图纸在列表中显示角标
- [ ] 完成确认后角标正确更新
