## 需求描述

### 背景与目标

设备管理员在工地现场完成设备加油后，需要快速记录加油信息。APP 端作为主要的数据录入端，需要提供简洁、高效的录入体验，支持离线场景下的数据暂存，并在联网后自动同步。

### 用户故事

```
作为 工地设备管理员
我想要 在完成设备加油后，立刻在手机 APP 上录入加油信息并拍照上传凭证
以便 管理层能在 PC 端实时查看油耗数据，我也不用再填纸质表单
```

## 功能需求

### 1. 入口位置

加油记录功能的入口位于首页 **Equipment（设备）** 模块卡片中：

- **首页** → **Equipment** 模块卡片 → 点击 **Refueling** 功能图标 → 进入加油记录列表页

> **注意**：Equipment 模块卡片与 Vehicle Delivery Booking 在首页同一层级，为独立的功能模块卡片。项目功能卡片（如 RWS Waterfront）中的 **Equipment List** 入口保持不变，仍指向设备列表页。

#### 2.2 列表卡片说明

每条加油记录以卡片形式展示，分为 **上下两个区域**：

**上部区域（设备信息区）— 左图右文布局**：
- **左侧**: 设备图片缩略图（正方形，圆角），无图片时显示设备类型默认图标
- **右侧**（从上到下共 4 行）:
  - **第一行**: 设备名称（粗体）
  - **第二行**: 设备编号（灰色副文本）
  - **第三行**: 设备类型（灰色副文本）
  - **第四行**: 所属分包商名称（灰色副文本，可能多行显示）

**下部区域（加油信息区）— 分隔线下方**：
- **第一行**: 加油日期 + 燃油类型标签（Diesel 蓝色、Gasoline 橙色、Other 灰色）
- **第二行**: 加油量（如 `150.00 L`）+ 总费用（如 `$277.50`，无费用则不显示）

#### 2.3 导航栏新增按钮

- 导航栏右侧显示 **[+]** 新增按钮（方形边框 + 加号图标）
- 点击 [+] → 进入新增加油记录页
- 仅 `equipment:refueling:create` 权限用户可见

#### 2.4 卡片交互

- 点击卡片 → 进入记录详情页
- 卡片右滑（或长按） → 显示「编辑」操作（仅 editable = true 的记录）
- 列表支持下拉刷新、上拉加载更多
- 空列表显示空状态插画 + 文案 "No refueling records yet"

#### 2.5 搜索与筛选交互

- **搜索栏**: 输入设备名称或编号，300ms 防抖后搜索
- **筛选图标**: 搜索栏右侧的漏斗图标，点击弹出筛选面板
- **筛选面板**: 底部弹出，包含以下筛选条件：
  - 燃油类型: All Types / Diesel / Gasoline / Other
  - 日期范围: 起始日期 ~ 结束日期
  - 底部操作按钮: Reset（重置） + Apply（应用）
- 筛选条件应用后自动刷新列表
- 有筛选条件激活时，漏斗图标显示角标（红点或数字）

#### 3.2 表单交互细节

**设备选择器**：
- 点击后弹出设备选择弹窗
- 弹窗顶部有搜索栏，可按设备名称/编号搜索
- 列表显示：设备名称 + 设备编号 + 设备类型 + 所属分包商
- 选中后显示设备名称，并自动填充设备编号和所属分包商

**日期选择器**：
- 使用 UNIAPP 原生日期选择器（`uni.showDatePicker` 或 `picker mode="date"`）
- 默认值为今天
- 最大可选日期为今天（不可选未来）

**时间选择器**：
- 使用 UNIAPP 原生时间选择器
- 可选字段，不填写时留空

**数字输入（加油量、单价）**：
- 点击输入框弹出数字键盘
- 支持小数点，最多两位小数
- 加油量输入后若已填单价，自动更新总费用
- 单价输入后若已填加油量，自动更新总费用

**总费用（只读）**：
- 灰色背景，不可手动编辑
- 当加油量和单价同时有值时自动计算并显示
- 任一为空时显示 "—"

**照片上传**：
- 点击 "+" 按钮调用 `uni.chooseImage` 选择照片
- 支持拍照和从相册选择
- 照片以缩略图展示，支持预览大图
- 点击已上传照片右上角 "×" 可移除
- 上传进度条显示（使用 `uni.uploadFile`）
- 照片压缩至 1080px 宽度后上传，减少流量消耗

**备注输入**：
- 多行文本域，最大 500 字
- 右下角显示实时字数统计 `${count}/500`

**提交按钮**：
- 蓝色渐变、胶囊形（全圆角），与登录页 Sign In 风格一致
- 提交时显示 Loading 状态，禁止重复点击
- 提交成功后显示 Toast "Record created successfully" 并返回列表页
- 提交失败后显示错误提示，保留表单数据

#### 3.3 表单校验交互

- 必填字段标签后显示 `*` 标记（红色）
- 用户点击提交时触发全量校验
- 校验失败的字段：输入框底部出现红色错误提示文本，页面自动滚动到第一个错误字段
- 用户修改字段值后实时清除该字段的错误提示
