## 需求描述

### 背景

[REQ-006-shared](../shared/REQ-006-shared.md) 定义了图纸版本审批通过后自动生成二维码并嵌入到 PDF。本需求定义**扫码后打开的 H5 公开状态页**，面向包括非系统用户在内的**所有现场人员**。

### 目标

1. 扫码用户打开**轻量 H5 页面**，无需登录或注册
2. 用户一眼判断手中图纸是否为**当前有效版本**（Active / Deprecated）
3. 对 Active 版本：若存在活跃局部更新，**提醒用户注意**
4. 对 Deprecated 版本：引导用户关注**最新版本信息**
5. **不展示图纸文件或附件**，仅展示状态元数据，防止知识产权外泄

### 用户故事

```
作为 现场任何查看图纸的人员（含非系统用户）
我想要 扫描图纸上的二维码立刻看到页面显示这张图纸是否有效
以便 不需要登录就能判断手中的图纸是否能放心施工
```

```
作为 现场 Site Engineer
我想要 扫码后看到该图纸版本是否有活跃的局部更新
以便 即使纸质图纸上没有标注，也能知道需要额外关注的改动
```

```
作为 持有已作废图纸的现场人员
我想要 扫码后明确看到"此版本已作废"的提示和最新版本信息
以便 立刻停止按此图施工，并向项目管理方索取最新版本
```

## 功能需求

### 1. 页面路由与数据加载

#### 1.1 路由规则

- URL: `https://{H5_HOST}/public/drawing-status/{publicToken}`
- 无需登录，任何用户访问均可看到内容
- 路径参数 `publicToken` 来自二维码中编码的字符串
- 页面加载时调用 `/public/drawing/status/{publicToken}`（见 [REQ-006-shared §4.1](../shared/REQ-006-shared.md)）

#### 1.2 加载状态

| 阶段 | 展示 |
|------|------|
| 请求中 | 居中 skeleton loader（模拟状态卡 + 信息行） |
| 请求超时（> 10s） | 错误提示 + `[Retry]` 按钮 |
| 请求失败（网络异常） | 错误提示 + `[Retry]` 按钮 |
| 返回 1003006001 | 无效 QR 错误页（§2.3） |
| 返回成功 | 渲染状态页（§2.1 或 §2.2） |

### 2. 页面布局

#### 2.1 ACTIVE 版本（当前有效）

```
┌──────────────────────────────────┐
│                                  │
│         Drawing Status           │   ← 页头
│                                  │
├──────────────────────────────────┤
│                                  │
│  ┌──────────────────────────┐    │
│  │  🟢  ACTIVE              │    │   ← 绿色状态徽章
│  │  · 3 Active Markups      │    │   ← 仅 count > 0 时显示
│  └──────────────────────────┘    │
│                                  │
│  Drawing Code    ARCH-001        │
│  Drawing Name    首层平面图       │
│  Category        Structural      │
│  Version         V2              │
│  Approved At     2026-04-10      │
│                                  │
│  ────────────────────────────    │
│                                  │
│  This drawing version is         │
│  currently in use.               │
│                                  │
│  ⚠️ There are 3 active markups.  │   ← 仅 count > 0 时显示
│  Please consult the design team  │
│  for the latest changes before   │
│  proceeding with construction.   │
│                                  │
└──────────────────────────────────┘
```

| 元素 | 规格 |
|------|------|
| 页头 | `Drawing Status` 文案；居中；18px；`--color-text-primary` |
| 状态徽章 | 绿色背景（`#E8F5E9` 底 / `#2E7D32` 文字），圆角 8px，内边距 12px 16px；图标 🟢 + 文字 `ACTIVE`，16px 粗体 |
| 徽章附加信息 | 若 `activeMarkupCount > 0`，在同一徽章内换行或以 `·` 分隔显示 `{n} Active Markups`（橙色 `#EF6C00`，14px） |
| 徽章附加信息（无 Markup） | `activeMarkupCount = 0` 时徽章内仅显示 `ACTIVE`，无附加文字 |
| 图纸信息区 | Label - Value 两列布局；Label 14px `--color-text-secondary`，Value 15px `--color-text-primary`；字段顺序固定：Code → Name → Category → Version → Approved At |
| 分隔线 | 浅灰色 `--color-border-light` |
| 底部主文案 | 固定文案 `"This drawing version is currently in use."`，15px |
| 底部警示文案 | 仅 `activeMarkupCount > 0` 时显示；橙色底 `#FFF3E0` + 橙色文字 `#EF6C00`；15px；圆角 8px；内边距 12px |

#### 2.2 DEPRECATED 版本（已作废）

```
┌──────────────────────────────────┐
│                                  │
│         Drawing Status           │
│                                  │
├──────────────────────────────────┤
│                                  │

│  │  🔴  DEPRECATED          │    │   ← 红色状态徽章
│  └──────────────────────────┘    │
│                                  │
│  Drawing Code    ARCH-001        │
│  Drawing Name    首层平面图       │
│  Category        Structural      │
│  Version         V1              │
│  Approved At     2026-03-20      │
│                                  │
│  ──── Superseded by ────         │   ← 分隔小标题
│                                  │
│  Latest Version  V3              │
│  Approved At     2026-04-15      │
│                                  │
│  ────────────────────────────    │
│                                  │
│  ⚠️ This version has been        │
│  superseded. Do NOT use this     │
│  drawing for construction.       │
│  Please obtain the latest        │
│  version from your project team. │
│                                  │
└──────────────────────────────────┘
```

| 元素 | 规格 |
|------|------|
| 状态徽章 | 红色背景（`#FFEBEE` 底 / `#C62828` 文字），其他规格与 Active 一致；**不显示**任何附加信息（即不显示 Markup 数量） |
| 已扫版本信息区 | 与 Active 布局一致，展示被扫描版本的 Code / Name / Category / Version / Approved At |
| Superseded by 分隔 | 居中文字 `Superseded by`，两侧短线；14px `--color-text-secondary` |
| 最新版本信息区 | 仅两个字段：`Latest Version`（版本号）+ `Approved At`（审批时间） |
| 底部警示文案 | 红色底 `#FFEBEE` + 红色文字 `#C62828`；15px；圆角 8px；内边距 12px |

#### 2.3 无效 Token 错误页

```
┌──────────────────────────────────┐
│                                  │
│          ❌                      │
│                                  │
│     QR Code Not Valid            │
│                                  │
│     The QR code on this drawing  │
│     is invalid or has expired.   │
│     Please contact your project  │
│     administrator.               │
│                                  │
└──────────────────────────────────┘
```

- 接口返回 1003006001 时渲染
- 不提供 `[Retry]` 按钮（token 无效通常为永久性问题）
- 不暴露任何项目 / 图纸信息

### 3. 交互规范

#### 3.1 只读展示
- 页面为**纯只读展示**，除错误页的 `[Retry]` 外**不提供任何交互按钮**
- 不展示图纸文件、附件预览、项目名称
- 不展示审批人、上传人等任何个人姓名信息
- 不提供登录入口、注册入口或跳转到 APP / PC 端的链接

#### 3.2 多语言

- 默认根据浏览器 `Accept-Language` Header 判断：
  - `zh-*` → 简体中文
  - 其他 → 英文
- 仅支持英文和简体中文两种语言
- 不依赖登录用户的语言偏好（无登录态）
- 以下字段的文案随语言切换：
  - 页头 `Drawing Status` / `图纸状态`
  - 状态徽章 `ACTIVE` / `有效`、`DEPRECATED` / `已作废`
  - `{n} Active Markups` / `{n} 条活跃局部更新`
  - 信息标签 `Drawing Code` / `图纸编号` 等
  - 所有底部文案
- `Category` 字段值遵循 [REQ-003-shared §2.6](../shared/REQ-003-shared.md) 的枚举对照，按语言显示（如 `Structural` / `结构`）

#### 3.3 响应式

- **移动端优先**设计（绝大多数访问来自手机扫码）
- 内容最大宽度 480px，在更宽屏上居中显示
- PC 浏览器访问相同 URL 时同样能正常显示，布局不变形
- 字号、行距按移动端可读性优先（正文 15px，徽章 16px）

### 4. 隐私与安全

| 项 | 要求 |
|----|------|
| 登录 | 页面**不加载**任何需要认证的资源，不显示登录入口 |
| Cookie / LocalStorage | 页面**不写入**任何 Cookie 或 LocalStorage |
| 用户追踪 | 不记录用户身份；后端日志仅记录 `publicToken + IP + 时间`，用于限流风控 |
| 敏感数据 | 不展示文件 URL、附件 URL、项目名称、人员姓名、内部 ID |
| 第三方脚本 | 禁用所有埋点、广告、第三方分析脚本（保持页面轻量与合规） |

## 验收标准

### ACTIVE 状态页
- [ ] 扫描 Active 版本的 QR 正确打开 H5 页面，显示绿色 `ACTIVE` 徽章
- [ ] 图纸信息区按顺序展示 Code、Name、Category、Version、Approved At 五个字段
- [ ] `activeMarkupCount > 0` 时徽章内显示 `· {n} Active Markups` 橙色附加信息
- [ ] `activeMarkupCount = 0` 时徽章内不显示附加信息
- [ ] 底部文案根据 Markup 数量正确切换（有 Markup 时显示橙色警示块；无 Markup 时仅显示中性文案）

### DEPRECATED 状态页
- [ ] 扫描已作废版本的 QR 显示红色 `DEPRECATED` 徽章
- [ ] 徽章内**不显示**局部更新数量
- [ ] 已扫版本信息区显示 Code、Name、Category、Version、Approved At
- [ ] `Superseded by` 分隔下方显示最新版本的 Version + Approved At
- [ ] 底部红色警示文案正确显示

### 错误与异常
- [ ] 无效 token 返回 1003006001 时渲染错误页，不暴露任何信息
- [ ] 错误页**不提供** `[Retry]` 按钮
- [ ] 网络超时 / 失败时显示错误提示和 `[Retry]` 按钮，点击后重新发起请求

### 隐私合规
- [ ] 页面不展示图纸文件、附件、项目名称
- [ ] 页面不展示任何人员姓名
- [ ] 页面无登录 / 注册入口，不引导用户下载 APP
- [ ] 不写入 Cookie 或 LocalStorage
- [ ] 无第三方埋点或分析脚本

### 响应式与多语言
- [ ] 移动端（≤ 480px）正常显示，所有信息完整可见
- [ ] PC 浏览器打开同一 URL 时页面居中展示，无布局错乱
- [ ] 英文 / 简体中文根据浏览器 `Accept-Language` 自动切换
- [ ] Category 英文名与中文名按语言正确显示
