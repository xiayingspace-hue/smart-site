## 需求描述

### 背景与目标

用户登录后进入管理后台，需要一套统一的页面框架（布局、导航、权限控制）承载所有业务功能模块。本文档定义管理后台的顶部导航栏（Header）、左侧侧边栏（Sidebar）、主内容区域（Main Content）的布局结构、样式规格、交互规范，以及会话管理在 PC 端的特有说明。具体业务页面内容在后续各功能需求中单独定义。

### 用户故事

```
作为 项目经理 / 系统管理员
我想要 登录后看到清晰的管理后台框架（导航、菜单、内容区）
以便 我可以快速找到并进入各功能模块
```

### 2. 通用交互规范

#### 2.1 页面路由与导航

- 点击侧边栏菜单项 → URL 路由变化 → 主内容区渲染对应页面
- 面包屑导航根据当前路由自动更新
- 浏览器前进/后退按钮正常工作
- 支持直接 URL 访问（需鉴权）

#### 2.2 权限控制

- 侧边栏菜单根据用户权限动态渲染（无权限的菜单不显示）
- 按钮级权限控制（如新增、编辑、删除按钮的显示/隐藏）
- 无权限访问的页面显示 403 提示页面
- 权限信息来自 `/system/auth/get-permission-info` 接口

#### 2.3 多语言支持

- 管理后台支持 中文（简体）/ English 切换
- 默认语言：English
- 语言切换入口：用户下拉菜单 或 Header 右侧独立入口
- 切换后所有文本即时刷新（菜单名称、面包屑、页面内容）
- 语言偏好保存到本地存储

#### 2.4 响应式适配

| 屏幕宽度 | 适配方案 |
|---------|---------|
| ≥ 1920px | 完整布局，侧边栏展开 |
| 1440-1919px | 完整布局，侧边栏展开 |
| 1280-1439px | 侧边栏默认折叠，可手动展开 |
| < 1280px | 提示用户使用更大屏幕，或侧边栏强制折叠 |

### 3. 会话管理

> 详见 [REQ-001-shared.md](../shared/REQ-001-shared.md) — 第 3 节「会话管理」

**PC 端特有说明**：
- **租户标识来源**: 从 URL 路径解析（如 `https://smartsite.mcc.sg/mcc/login` → `tenantCode = mcc`），登录成功后保存 `tenantId` 到本地
- Token 存储在 `localStorage` 或 `sessionStorage`（根据 "Remember the password" 选择）
- 选中 "Remember the password" → 使用 `localStorage`（关闭浏览器后保留）
- 未选中 → 使用 `sessionStorage`（关闭浏览器后清除）
- 所有 API 请求通过 Axios 拦截器统一添加 Authorization、X-Tenant-Id、Project-Id、lang Header
- `X-Tenant-Id` 的值来自登录成功后后端返回的 `tenantId`（非 URL 中的 `tenantCode`）
