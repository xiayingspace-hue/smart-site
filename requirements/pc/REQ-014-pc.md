## 3. 问题分类

### 3.1 REQ-003 工程图纸管理反馈

#### FB-001 上传新版本时支持修改 Drawing Code

**背景：**
图纸在修版过程中，图纸编号（Drawing Code）本身有时会随之变更（如专业代码、区域代码调整），导致新版图纸文件内标注的 Drawing Code 与系统中记录的编号不一致。当前系统在上传新版本时不支持修改 Drawing Code，用户无法保持系统编号与实物图纸一致。

**期望行为：**
- 上传图纸新版本时，弹框中的 Drawing Code 字段支持用户手动修改
- 修改后的 Drawing Code 仅对该新版本生效，历史版本的 Drawing Code 记录不受影响
- Drawing Code 在同一项目下仍需满足唯一性校验（修改后与其他图纸编号不冲突）
- 历史版本的 Drawing Code 在版本记录中保留原值，方便追溯

#### FB-002 图纸版本附件（Attachments）支持

**背景：**
用户希望在图纸版本下能上传除PDF以外的相关文件（如CAD、计算书、施工说明等），SE可在查阅图纸时一并下载所需附件，提升现场工作效率。

**期望行为：**
- 图纸版本详情页新增"附件（Attachments）"区域，支持上传任意类型文件（.dwg、.dxf、.docx、.xlsx、图片等）
- 附件为可选，支持多个附件，列表展示（附件名称/类型/大小/上传时间/操作）
- 上传时可填写附件备注，便于区分用途
- 附件与具体版本绑定，不同版本的附件独立管理
- 创建新版本时可同步上传附件，版本创建后也可在详情页随时补充或删除
- 上传/删除附件权限与PDF上传人一致
- SE下载附件权限与PDF下载权限一致（已分配SE可下载）
- 不影响PDF主文件的审批、QR码、SE分配等主流程

### 3.2 REQ-004 图纸局部更新（Markup）反馈

#### FB-003 "+Markup"入口移入Markup历史弹框

**背景：**
当前图纸列表操作列中存在多个按钮（如查看、审批、+Markup等），随着功能增加按钮数量较多，界面显得拥挤，用户希望精简操作列。

**期望行为：**
- 将图纸列表操作列中的"+Markup"按钮移除，不再在操作列直接展示
- 操作列保留"查看Markup历史"入口（如图标按钮），点击后弹出Markup历史记录弹框
- 在Markup历史弹框的顶部或底部新增"+ Add Markup"按钮，作为发布新Markup的入口
- 功能本身不变，仅调整入口位置，用户需先进入历史弹框再发布新Markup
- 操作列按钮数量减少，界面更简洁

#### FB-004 Markup列表显示对应图纸编号（Drawing Code）

**背景：**
Markup 历史记录列表中，用户希望能直接看到每条 Markup 对应的图纸编号（Drawing Code），以便快速识别该 Markup 属于哪张图纸，无需逐条打开查看。

**期望行为：**
- Markup 创建时，系统自动记录当前图纸的 Drawing Code，无需用户手动填写
- Markup 历史记录列表中新增"图纸编号"列，展示该条 Markup 对应的 Drawing Code
- 该字段只读，创建后不可修改

#### FB-005 SE 不可见 Markup（权限收回）

**背景：**
局部更新（Markup）通常发生在图纸内部审批之前，是上传方、PM、CM 之间的内部修改沟通动作。图纸审批通过后才正式分发给 SE。因此 SE 不应介入 Markup 流程，也不应看到相关内容。

**期望行为：**
- SE 角色在 PC 端和 APP 端图纸页面中，完全不展示 Markup 相关入口（包括"查看 Markup 历史"按钮、Markup 数量角标等）
- SE 无法通过任何路径访问 Markup 历史记录及 Markup 文件内容
- 对 REQ-005（SE PC端图纸访问）中涉及 Markup 的展示逻辑进行移除

### 3.3 REQ-005 SE PC端图纸访问反馈

#### FB-006 SE视角隐藏系统版本号（Ver），仅显示 Drawing Code

**背景：**
系统对每张图纸同时维护 Drawing Code（图纸编号）和系统版本号（Ver，系统自动递增）。SE 在查看图纸时不需要了解系统内部版本序号，只需通过 Drawing Code 识别图纸即可。显示系统 Ver 会引入额外信息，增加 SE 的认知负担。

**期望行为：**
- SE 角色在 PC 端和 APP 端查看图纸时，**只显示 Drawing Code**，不显示系统版本号（Ver）字段
- 图纸管理列表（图纸上传人员 / PM / CM 视角）中，**同时显示 Drawing Code 和系统 Ver**，供内部版本管理和追踪使用
- 系统 Ver 字段的显示隐藏逻辑由角色权限控制，SE 不感知系统 Ver 的存在
