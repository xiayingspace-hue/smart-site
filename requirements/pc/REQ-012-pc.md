# REQ-012 CM"我的任务"菜单与任务执行页面需求

## 1. 需求背景

为提升施工经理（CM）在项目进度管理系统中的工作效率和体验，需要为CM提供专属的"我的任务"菜单和页面，便于其聚焦自身任务执行、下级分配和问题反馈。

## 2. 需求描述

作为施工经理（CM），
我希望登录系统后，点击左侧Progress Management下的"我的任务（CM）"菜单，可以：
- 查看PM分配给我的所有任务（仅显示与我相关的任务）。
- 通过任务列表操作列将任务分解/分配给SE。
- 针对任务进展中的实际问题，直接向PM汇报。

## 3. 页面结构与交互

### 3.1 我的任务菜单（CM专用）
- 入口：Progress Management > 我的任务（CM）
- 行为：点击后进入"我的任务（CM）"页面，仅CM可见。

#### 3.1.1 页面功能
1. 任务列表：展示所有分配给当前CM的任务，支持筛选、搜索、分页。
2. 任务详情：点击任务行进入任务详情页（只读），页面顶部以 Tab 方式组织不同类型信息，当前版本包含以下 Tab：
   - **任务信息**：展示当前任务的基本字段（Activity ID、Activity Name、WBS、计划/实际时间、Deviation、Status、优先级、备注等）
   - **工序任务**：展示CM已创建的所有工序任务列表，列表字段包含：工序名称、权重、计划时间、实际开始/结束时间（系统自动记录，只读）、优先级、负责SE、状态（Not Started / In Progress / Completed）；支持点击某条工序任务的"编辑"按钮，从页面右侧以侧滑弹框方式进行修改（字段与创建时一致：任务名称、计划时间、优先级、负责SE、备注等）；**Completed 状态行整体置灰，编辑按钮 disabled**；修改保存后列表自动刷新
   - **问题上报**：展示SE在该任务下上报的所有问题记录及处理状态（只读）
   - *(预留)* 后续可在此扩展"关联图纸"、"关联物料"等 Tab，结构不变
3. 工序任务：在任务列表操作列点击"新建工序任务"按钮，从页面右侧以侧滑弹框方式进入工序任务创建流程；每条工序任务在创建时直接指定负责的 SE。
4. 问题汇报：在任务列表操作列点击"汇报问题"按钮，弹窗填写问题类型、描述、影响、建议、附件等，提交后自动通知PM。

#### 3.1.2 页面结构
- 页面整体布局与PM端"我的任务"页面保持一致：
   - 顶部为筛选/搜索区，支持按任务状态、时间、WBS等筛选
   - 下方为任务表格，字段包括：Activity ID、Activity Name、WBS、计划/实际时间、Deviation、Status、Assigned SE、操作等
   - 表格样式、分页、批量操作、筛选交互等与PM端一致
   - 操作列包含：**新建工序任务**、**汇报问题** 两个按钮，显示/置灰规则如下：

     | 按钮 | 显示条件 | 置灰条件 |
     |------|---------|---------|
     | 新建工序任务 | 仅 CM 角色可见 | 任务状态为"已完成"或"已作废"时置灰，不可点击 |
     | 汇报问题 | 仅 CM 角色可见 | 任务状态为"已作废"时置灰；其余状态均可点击 |

     > **补充说明：**
     > - 任务状态为"进行中"、"待开始"、"逾期"时，"新建工序任务"按钮**可点击**
     > - 任务已有工序任务时，仍可继续点击"新建工序任务"追加新工序（无数量上限）
     > - 工序任务的数量上限如有业务需要，可在后续迭代中引入

   - 点击任务行进入任务详情页，详情页顶部以 Tab 方式展示：**任务信息** / **工序任务** / **问题上报**，预留后续扩展 Tab 位置；任务信息和问题上报为只读，工序任务支持编辑
- 工序任务创建侧滑弹框（由操作列"新建工序任务"触发）：
   - 弹框顶部可选择"引用工序模板"或"自定义新建"
   - 选择"引用工序模板"时，弹出模板选择区，支持按专业、工序类型、区域等筛选，批量勾选后自动生成工序任务行；每行权重自动带出模板预设值，CM 可修改
   - 选择"自定义新建"时，可手动添加工序任务行；权重字段为必填，由 CM 手动输入
   - 每行支持批量编辑，可批量保存，所有工序任务自动与上级任务关联
- 侧滑弹框字段清单：
   - 关联上级任务（自动带出当前行任务，只读）
   - 工序模板选择入口
   - 工序任务名称
   - WBS
   - 计划开始/结束时间
   - 优先级
   - 权重（引用模板时自动带出，可修改；自定义时必填；输入范围建议 1～10）
   - 分配 SE
   - 备注
   - 批量添加/编辑/删除工序任务行

#### 3.1.3 消息通知
- CM分配任务给SE后，SE收到"新任务分配"通知
- CM汇报问题后，PM收到"问题汇报"通知，CM可收到处理反馈
- 其它通知场景（如变更、作废、进度反馈、逾期预警、退回、资料补充、审批、评论、完成确认、分解合并等）同PM端

#### 3.1.4 权限说明
- "我的任务（CM）"菜单仅CM可见，SE仅能看到被分配的子任务，PM不可见

#### 3.1.5 任务状态说明
- 状态定义、判定规则、展示方式与PM端一致

#### 3.1.6 工序进度与任务进度计算规则

**工序任务状态流转**

工序任务采用三态模型，状态单向流转，由 SE 在 APP 端操作驱动：

```
Not Started  ──（SE 点击 Start Task）──→  In Progress  ──（SE 点击 Mark as Complete）──→  Completed（锁定）
```

| 状态 | 说明 | 实际时间记录 |
|------|------|------------|
| Not Started | 工序任务创建后的初始状态 | 实际开始/结束时间均为"—" |
| In Progress | SE 点击 Start Task 后进入 | 系统自动记录**实际开始时间** |
| Completed | SE 点击 Mark as Complete 确认后进入，**锁定** | 系统自动记录**实际结束时间** |

**工序完成状态（SE 操作）**
- SE 在 APP 端操作按钮均使用英文显示：**Start Task** / **Mark as Complete** / **Report Issue**
- SE 点击 **Mark as Complete** 时，系统弹出确认提示：*"标记完成后将锁定该工序，如需修改请联系 CM，确认继续？"*，SE 确认后方可提交
- SE 标记完成后状态变为 Completed 并**直接生效锁定**，SE 自身无法撤销；如需修改完成状态，须由 CM 在任务详情"工序任务" Tab 中操作解锁
- *(待规划)* CM 解锁工序完成状态的具体流程（含解锁原因填写、SE 通知、日志记录）将在后续迭代中引入
- SE 无需填写进度百分比，系统根据 Completed 状态自动计算

**任务进度计算公式**

$$
任务进度 = \frac{\sum(\text{状态为 Completed 的工序权重})}{\sum(\text{全部工序的权重})} \times 100\%
$$

**权重来源规则**

| 工序创建方式 | 权重来源 | CM 是否可修改 |
|-------------|----------|--------------|
| 引用工序模板 | 自动带出模板预设值 | ✅ 可修改 |
| 自定义新建 | CM 手动填写（必填） | ✅ 可修改 |

**示例**

| 工序任务 | 权重 | 状态 |
|----------|------|------|
| 土方开挖 | 3 | ✅ Completed |
| 基础浇筑 | 5 | ✅ Completed |
| 墙体砌筑 | 2 | 🔲 In Progress |

$$
任务进度 = \frac{3 + 5}{3 + 5 + 2} \times 100\% = 80\%
$$

**注意事项**
- 权重为相对值，建议范围 1～10，无需归一化
- 任务进度由系统实时计算，CM 和 PM 只读，不可手动填写
- 工序任务权重修改后，任务进度自动重新计算
- **实际开始/结束时间**由系统在 SE 操作时自动记录，CM 在工序任务 Tab 中只读查看

## 4. 验收标准

- [ ] CM可在"我的任务（CM）"页面查看所有被分配任务
- [ ] CM可通过操作列"新建工序任务"按钮创建工序任务并指定负责 SE
- [ ] 任务详情页顶部以 Tab 展示：任务信息（只读）、工序任务（支持编辑）、问题上报（只读）
- [ ] CM可针对任务进展中的问题向PM汇报，并可追溯汇报历史
- [ ] SE和PM可收到相应的消息通知
- [ ] 权限控制符合角色要求

## 5. 对其他需求的影响

| 需求 | 影响说明 |
|------|----------|
| **REQ-013**（SE APP 端） | SE 需在 APP 端查看分配给自己的工序任务列表，支持 **Start Task**（Not Started→In Progress，自动记录实际开始时间）和 **Mark as Complete**（In Progress→Completed，自动记录实际结束时间，锁定）操作 |
| **REQ-005**（SE PC 端） | SE 需在 PC 端同样可查看分配给自己的工序任务列表（只读）；PC 端不提供"标记完成"操作，该操作仅限 APP 端 |

> 备注：本需求为CM专属任务执行与问题汇报功能，配合PM端实现全流程进度管理。
