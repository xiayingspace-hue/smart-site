## 基本信息

- **需求ID**: REQ-004-pc

## 需求描述

### 背景与目标

设计人员需要在 PC 端对已发布的图纸版本进行局部补充说明，无需走完整审批流程，同时精准通知相关的 Site Engineer。当多次局部更新积累后，可一键汇总提交为正式新版本。

### 用户故事

```
作为 图纸设计人员
我想要 在 PC 端为某张图纸发布局部更新，并附上说明和补充文件
以便 现场工程师能快速了解该图纸的局部改动，无需等待完整版本发布
```

```
作为 图纸设计人员
我想要 在 PC 端查看某图纸的所有局部更新，并选择将其汇总为新版本
以便 在改动积累到一定程度后，发起正式版本审批，形成完整的版本记录
```

## 功能需求

### 1. 入口位置

局部更新功能集成在 **REQ-003 图纸列表页** 的表格操作列中，新增操作按钮：

| 按钮 | 显示条件 | 权限 | 说明 |
|------|---------|------|------|
| Markups | 始终显示 | `drawing:view` | 打开局部更新抽屉，查看该图纸的所有局部更新 |
| + Markup | `status = ACTIVE` | `drawing:markup:publish` | 打开发布局部更新弹窗 |

### 2. 图纸列表页调整

#### 2.1 表格操作列变化

在 REQ-003 图纸表格的 Actions 列新增以下按钮（插入在 Assign 按钮之后）：

```
[View] [History] [Confirms] [Assign] [Markups] [+ Markup]  [Upload V{n}]
```

> `[+ Markup]` 仅在 `status = ACTIVE` 时展示，`status = PENDING_APPROVAL` 或 `REJECTED` 时隐藏。

#### 2.2 Markups 列（可选）

在图纸表格中新增可选列 **Markups**，显示该图纸当前 `ACTIVE` 状态的局部更新数量：

| 列名 | 字段 | 宽度 | 说明 |
|------|------|------|------|
| Markups | `activeMarkupCount` | 90px | `ACTIVE` 数量；0 时显示 `—`；点击打开局部更新抽屉 |

---

### 3. 发布局部更新弹窗

#### 3.1 触发方式

点击表格操作列 **[+ Markup]** 按钮，弹出发布局部更新弹窗。

#### 3.2 弹窗布局

```
┌────────────────────────────────────────────────────────┐
│  Publish Markup — ARCH-001 · 首层平面图  V3      [✕]  │
├────────────────────────────────────────────────────────┤
│                                                        │
│  Title *                                               │
│  [__________________________________________________]  │
│                                                        │
│  Description *                                         │
│  [__________________________________________________]  │
│  [                                                  ]  │
│  [                                                  ]  │
│                                                        │
│  Affected Area                                         │
│  [__________________________________________________]  │
│  e.g. "A-C axis / Level 3-5"                          │
│                                                        │
│  Attachments（最多 10 个，每个 ≤ 20MB）                 │
│  ┌────────────────────────────────────────────────┐    │
│  │  📎  Drag & Drop or  [Browse Files]            │    │
│  │  Supports: PDF / PNG / JPG · Max 20MB each     │    │
│  └────────────────────────────────────────────────┘    │
│  ┌──────────────────────────────────────────────┐      │
│  │  node-detail.pdf  (512KB)              [✕]   │      │
│  └──────────────────────────────────────────────┘      │
│                                                        │
├────────────────────────────────────────────────────────┤
│                         [Cancel]  [Publish]            │
└────────────────────────────────────────────────────────┘
```

#### 3.3 表单字段规格

| 字段 | 组件 | 校验规则 |
|------|------|---------|
| Title | el-input | 必填，≤200 字符 |
| Description | el-input type="textarea" rows=4 | 必填，≤2000 字符，右下角字数计数 |
| Affected Area | el-input | 可选，≤500 字符，placeholder 示例文字 |
| Attachments | 自定义多文件上传区 | 可选；格式 PDF/PNG/JPG；单文件 ≤20MB；最多 10 个 |

#### 3.4 发布交互

1. 点击 [Publish] 前端校验必填项
2. 附件逐个上传，展示每个文件的上传进度
3. 全部附件上传完成后调用 `/drawing/markup/publish` 接口
4. 发布成功后：
   - 弹窗关闭
   - 图纸表格 `Markups` 列数字 +1
   - 顶部 Toast：`"Markup published. Assigned Site Engineers have been notified."`
5. 已填写内容时点击取消或遮罩，弹出二次确认

### 4. 局部更新列表抽屉

#### 4.1 触发方式

点击表格操作列 **[Markups]** 按钮，从右侧滑入局部更新列表抽屉。

#### 4.2 抽屉布局

```
┌──────────────────────────────────────────────────────────┐
│ ← Markups — ARCH-001 · 首层平面图                         │
│   Based on V3 · 2 Active  1 Merged                       │
├──────────────────────────────────────────────────────────┤
│  [All]  [Active (2)]  [Merged (1)]         [+ Markup]    │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  ┌──────────────────────────────────────────────────┐   │
│  │  [Active]  A轴节点详图修正                         │   │
│  │  Apr 8, 2026 · 张三                               │   │
│  │  A-C轴 / 3-5层                                   │   │
│  │  A轴与3轴交叉节点详图已更新，新增钢筋排布说明...    │   │
│  │  📎 node-detail.pdf                              │   │
│  │                         [☑ Merge]  [🗑 Delete]  │   │
│  └──────────────────────────────────────────────────┘   │
│                                                          │
│  ┌──────────────────────────────────────────────────┐   │
│  │  [Active]  C区消防管道路由修正                     │   │

│  │                         [☑ Merge]  [🗑 Delete]  │   │

│                                                          │
│  ┌──────────────────────────────────────────────────┐   │

│  │  Apr 5, 2026 · 张三                               │   │
│  └──────────────────────────────────────────────────┘   │
│                                                          │

│  └────────────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────────────┘
```

#### 4.3 抽屉属性

| 属性 | 规格 |
|------|------|
| 组件 | `el-drawer`，direction="rtl" |
| 宽度 | 560px |
| 标题 | "Markups — {drawingCode} · {drawingName}" |
| 副标题 | "Based on {baseVersionNo} · {activeCount} Active  {mergedCount} Merged" |
| Tab 筛选 | All / Active ({n}) / Merged ({n})，`el-tabs` |
| [+ Markup] 按钮 | 抽屉右上角，复用发布局部更新弹窗 |
| 底部汇总栏 | 有局部更新被勾选时固定显示，展示已选数量 + [Merge to New Version →] 按钮 |

#### 4.4 局部更新卡片

| 属性 | 规格 |
|------|------|
| 状态标签 | Active（绿）/ Merged → V{n}（灰，含目标版本号） |
| 标题 | 14px，`--font-weight-semibold` |
| 元信息行 | "日期 · 创建人"，12px，`--color-text-secondary` |
| 影响区域 | 灰色 tag 样式展示 |
| 说明文字 | 最多显示 3 行，超出展示"...Show more" |
| 附件列表 | 📎 图标 + 文件名，可点击预览 |
| [☑ Merge] 复选框 | 仅 `ACTIVE` 状态卡片显示；勾选后计入底部汇总栏 |
| [🗑 Delete] | 仅创建人可见，`ACTIVE` 状态；点击二次确认后删除 |

---

### 5. 汇总为新版本

#### 5.1 触发方式

在局部更新抽屉中勾选一条或多条 `ACTIVE` 局部更新后，点击底部 **[Merge to New Version →]** 按钮，打开汇总弹窗。

#### 5.2 汇总弹窗布局

```
┌────────────────────────────────────────────────────────┐
│  Merge Markups to New Version — ARCH-001       [✕]    │
├────────────────────────────────────────────────────────┤
│                                                        │
│  Selected Markups (2):                                 │
│  ┌────────────────────────────────────────────────┐   │
│  │  ☑ A轴节点详图修正        Apr 8, 2026          │   │
│  │  ☑ C区消防管道路由修正    Apr 7, 2026          │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
│  New Version File *                                    │
│  ┌────────────────────────────────────────────────┐   │
│  │  📁  Drag & Drop or  [Browse Files]            │   │
│  │  Supports: PDF / PNG / JPG · Max 50MB          │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
│  ℹ️  Current version: V3. After approval, this will   │
│     become V4.                                         │
│                                                        │
│  Version Note                                          │
│  [Merged from 2 markups: A轴节点详图修正; C区消防...]  │
│  （自动填充，可手动修改）                               │
│                                                        │
│  Approver *                                            │
│  [Select approver...         ▼]                        │
│                                                        │
├────────────────────────────────────────────────────────┤
│                         [Cancel]  [Submit for Approval]│
└────────────────────────────────────────────────────────┘
```

#### 5.3 汇总弹窗字段规格

| 字段 | 规格 |
|------|------|
| Selected Markups | 只读，展示已勾选的局部更新列表（标题 + 日期） |
| New Version File | 必填，格式 PDF/PNG/JPG，≤50MB |
| 版本提示 | 蓝色信息块，显示当前版本和将升级的版本号（只读） |
| Version Note | el-input，预填 "Merged from {n} markups: {titles}"，可手动修改，≤500 字符 |
| Approver | el-select filterable，必填，从人员接口拉取审批人列表 |

#### 5.4 提交后处理

1. 调用 `/drawing/markup/merge` 接口
2. 成功后：
   - 弹窗关闭，局部更新抽屉关闭
   - 图纸表格刷新：状态变为 `Pending`，Markups 列数字减少（已汇总条目不再计入 Active 数）
   - Toast：`"Submitted for approval. Version note includes {n} markups."`
3. 若该图纸有版本处于 `PENDING_APPROVAL`，提交时接口返回错误，前端显示 `el-alert`：`"A version of this drawing is already pending approval. Please wait for the review to complete."`

## 验收标准

### 发布局部更新
- [ ] 点击 [+ Markup] 打开发布弹窗，标题中正确显示图纸编号、名称和当前版本号
- [ ] Title / Description 为空时提示必填，[Publish] 不可点击
- [ ] 附件格式非 PDF/PNG/JPG 时显示格式错误提示
- [ ] 单个附件超过 20MB 时显示大小错误提示
- [ ] 附件数量达到 10 个后禁止继续添加
- [ ] 发布成功后表格 Markups 列 +1，Toast 正确显示
- [ ] 有内容时取消弹出二次确认

### 局部更新列表抽屉
- [ ] 打开抽屉正确显示该图纸下所有局部更新，Active / Merged 分 Tab 展示
- [ ] Active 状态卡片显示 [☑ Merge] 复选框和 [🗑 Delete] 按钮（仅创建人可见删除）
- [ ] Merged 状态卡片显示目标版本号（如 "Merged → V4"），无操作按钮
- [ ] 勾选局部更新后底部汇总栏出现，显示已选数量
- [ ] 全部取消勾选后底部汇总栏隐藏

### 删除局部更新
- [ ] 点击删除弹出二次确认，确认后从列表移除，Markups 列数字 -1
- [ ] Merged 状态的局部更新无删除按钮
- [ ] 非创建人无删除按钮

### 汇总为新版本
- [ ] 汇总弹窗显示已选局部更新列表（只读）
- [ ] Version Note 自动预填汇总摘要，可手动修改
- [ ] 版本提示正确显示当前版本号和下一版本号
- [ ] New Version File 和 Approver 为必填
- [ ] 汇总成功后所选局部更新状态变为 Merged，图纸状态变为 Pending
- [ ] 若该图纸已有版本待审批，提交被拒绝并显示错误提示
