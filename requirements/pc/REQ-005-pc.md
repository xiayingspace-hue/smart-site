## 需求描述

### 背景与目标

Site Engineer 目前通过 APP 接收图纸通知并完成查阅确认，但在办公室或使用电脑时切换到手机查看图纸体验较差，且大尺寸图纸（如 A1 平面图）在 PC 大屏上查看更清晰高效。

本需求目标：
1. Site Engineer 可在 PC 端访问与 APP 等效的图纸列表（仅显示分配给自己的图纸）
2. 在 PC 端在线查看图纸并完成"查阅确认（Confirm Reading）"，确认记录跨端同步
3. 在 PC 端查看图纸的所有 ACTIVE 局部更新（Markups），并完成"标记已读（Mark as Read）"
4. 在 PC 端接收图纸局部更新的站内通知（In-App Notification），点击跳转到对应图纸的 Markups 面板

### 用户故事

```
作为 Site Engineer
我想要 在 PC 端看到分配给我的图纸列表
以便 在使用电脑时也能方便地访问相关图纸，不必切换到手机
```

```
作为 Site Engineer
我想要 在 PC 端在线查看图纸并点击"Confirm Reading"完成查阅确认
以便 在大屏上看清楚图纸细节，并留存跨平台的阅读记录
```

```
作为 Site Engineer
我想要 在 PC 端查看某张图纸的所有局部更新（Markups）并逐条标记已读
以便 在使用电脑时也能掌握图纸的最新改动，不遗漏任何局部更新
```

```
作为 Site Engineer
我想要 在 PC 端收到图纸局部更新的站内通知
以便 在使用电脑时也能第一时间获知图纸改动，无需主动切换到手机
```

## 功能需求

### 1. 入口与权限

#### 1.1 菜单入口

- Site Engineer 登录 PC 后，侧边栏显示 **Drawings** 菜单项（与管理员的 Drawing Management 菜单不同）
- 若当前用户角色同时包含管理员角色，则 Drawings 与 Drawing Management 均显示，分属不同菜单项

#### 1.2 数据过滤规则

与 REQ-003-app §2.2 保持一致：
- 仅显示 `status = ACTIVE` 的图纸
- 仅显示已被管理员**分配**给当前 Site Engineer 的图纸（DrawingAssignment）
- 后端双重过滤，前端无法绕过

---

### 2. Site Engineer 图纸列表页

#### 2.1 页面布局

```
┌─────────────────────────────────────────────────────────┐
│ 📄 Drawings                        [🔍 Search...]       │
├─────────────────────────────────────────────────────────┤
│ Code / Name     │ Category  │ Version │ Status    │ Markups │ Confirmed │ Updated     │
│─────────────────────────────────────────────────────────│
│ ARCH-001        │ Arch.     │ V3      │ ✓ Read   │  ●2    │ Apr 8     │ 2026-04-01  │
│ 首层平面图       │           │         │           │         │           │             │
│─────────────────────────────────────────────────────────│
│ STRU-001        │ Struct.   │ V2      │ Confirm → │  —      │ —         │ 2026-03-20  │
│ 基础结构图       │           │         │           │         │           │             │
└─────────────────────────────────────────────────────────┘
```

- 顶部搜索框（Code 或 Name 模糊搜索）+ Category 下拉筛选，参考 REQ-003-pc §2.2 样式
- 无需 Upload Drawing / Manage 相关操作，本页仅只读查阅功能

#### 2.2 表格列定义

| 列名 | 说明 |
|------|------|
| Drawing Code / Name | 图纸编号（加粗）+ 名称（次级文字） |
| Category | 分类标签 |
| Version | 当前版本号（如 V3） |
| Status | 当前用户对当前版本的确认状态：`✓ Read`（绿）/ `Confirm →`（蓝色文字链接，点击跳转在线查看页） |
| Markups | 该图纸的 ACTIVE 局部更新未读数量：`●{n}`（橙色，n>0 时显示）/ `—`（无未读） |
| Confirmed | 当前用户完成确认的时间（`Apr 8`），未确认时显示 `—` |
| Last Updated | 图纸最后更新时间 |

#### 2.3 空状态

- 尚未被分配任何图纸时：居中显示图纸图标 + "No drawings assigned to you yet."

### 3. 图纸在线查看页

#### 3.1 触发方式

点击表格行任意区域，或点击 `Confirm →` 状态链接，进入在线查看页（独立页面或 El-Dialog 抽屉，由 UI 设计决定）。

#### 3.2 页面内容

```
┌────────────────────────────────────────────────────────────┐
│  ← Drawings  /  ARCH-001 首层平面图                        │  ← 面包屑
├────────────────────────────────────────────────────────────┤
│  ARCH-001  首层平面图  ·  V3  ·  🟢 Active                 │
│  Architectural  ·  Updated 2026-04-01                       │
├────────────────────────────────────────────────────────────┤
│  [Drawing Preview]  [Markups (2)]                          │  ← 顶部 Tab
├────────────────────────────────────────────────────────────┤
│                                                            │
│          （图纸在线预览区域，支持缩放 / 平移）               │
│                                                            │
├────────────────────────────────────────────────────────────┤
│  V3  ·  Updated 2026-04-01                                 │
│  ┌─────────────────────────────────┐                       │
│  │  ✓  Confirm Reading             │                       │
│  └─────────────────────────────────┘                       │
└────────────────────────────────────────────────────────────┘
```

#### 3.3 在线预览规范

- 遵循 REQ-003-app §3.3 预览规则（PDF / PNG / JPG / DWG→PDF 转换）
- PC 端支持鼠标滚轮缩放、鼠标拖拽平移
- 提供"全屏查看"按钮（F11 或页面内全屏）

#### 3.4 Confirm Reading 操作

| 状态 | 显示 | 操作 |
|------|------|------|
| 未确认 | `[Confirm Reading]` 主按钮 | 点击弹出确认对话框，确认后调用 `/drawing/confirm` |
| 已确认 | `✓ Confirmed on {date}`（绿色文字，不可点击） | 显示确认时间 |

- 确认成功后列表页中该图纸的 Status 列自动更新为 `✓ Read`
- 确认记录 `deviceInfo` 字段记录为 `PC`，与 APP 端确认记录统一汇总

### 4. Markups Tab — 局部更新列表

#### 4.1 触发方式

在图纸在线查看页顶部切换到 **Markups** Tab，或从图纸列表点击 `●{n}` 角标直接进入 Markups Tab。

#### 4.2 布局

```
┌────────────────────────────────────────────────────────────────┐
│  [Drawing Preview]  [Markups (2)]                              │
├────────────────────────────────────────────────────────────────┤
│  ┌──────────────────────────────────────────────────────────┐  │
│  │  [New]  A轴节点详图修正                     Apr 8, 2026  │  │
│  │  Published by 张三                                        │  │
│  │  Affected Area: A-C轴 / 3-5层                             │  │
│  │  A轴与3轴交叉节点详图已更新，新增钢筋排布说明。            │  │
│  │  📎 node-detail.pdf                                       │  │
│  │                                    [✓ Mark as Read]       │  │
│  └──────────────────────────────────────────────────────────┘  │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │  C区消防管道路由修正                        Apr 7, 2026  │  │
│  │  Published by 张三                                        │  │
│  │  Affected Area: C区 / B1层                                │  │
│  │  消防主管道路由变更，详见附图...                           │  │
│  │  📎 route-update.png                                      │  │
│  │                                             ✓ Read        │  │
│  └──────────────────────────────────────────────────────────┘  │
└────────────────────────────────────────────────────────────────┘
```

#### 4.3 局部更新卡片规格

- 仅显示 `status = ACTIVE` 的局部更新；已汇总（MERGED）的不显示
- 按 `publishTime` 降序排列
- 每张卡片展示：标题、发布人、发布时间、影响区域、说明文字（最多 3 行，超出省略）、附件列表
- `[New]` 标签：橙色，当前用户未确认时显示
- `[Mark as Read]` 按钮：当前用户未确认时显示；点击调用 `/drawing/markup/confirm`；成功后替换为 `✓ Read`（绿色）

#### 4.4 附件查看

- 点击附件文件名或 [Preview] 链接，在浏览器新 Tab 打开附件
- 支持 PDF 内嵌预览，PNG / JPG 直接打开

### 5. PC 端站内通知

#### 5.1 通知来源

当管理员在 PC 端发布局部更新后，系统除发送 App Push 通知外，同步向同一目标用户群（已被分配该图纸的 Site Engineer）发送 **PC 端站内通知（In-App Notification）**。

#### 5.2 通知中心入口

- PC 顶部导航栏右侧显示 🔔 铃铛图标
- 有未读通知时图标上显示红色数字角标（最多显示 99，超出显示 99+）
- 点击铃铛打开通知下拉面板（最近 20 条）

#### 5.3 通知格式

| 字段 | 内容 |
|------|------|
| 图标 | 📄 图纸图标（橙色） |
| 标题 | Drawing Update — {drawingCode} |
| 正文 | {markupTitle}. Please review the latest markup for {drawingCode} {drawingName}. |
| 时间 | 通知发送时间（如 "5 minutes ago" / "Apr 8"） |
| 跳转 | 点击通知条目 → 跳转到对应图纸在线查看页，并自动激活 **Markups Tab** |

#### 5.4 通知已读逻辑

- 点击通知条目后，该通知标记为已读，角标数量减少
- 通知下拉面板顶部提供 "Mark all as read" 操作

## 验收标准

### 图纸列表页
- [ ] Site Engineer 登录 PC 后侧边栏可见 Drawings 菜单
- [ ] 列表仅显示 `status = ACTIVE` 且已被分配给当前 Site Engineer 的图纸
- [ ] 未被分配任何图纸时显示空状态文案
- [ ] Status 列正确反映当前用户的确认状态（`✓ Read` / `Confirm →`）
- [ ] Markups 列正确显示 ACTIVE 未读数量；无未读时显示 `—`

### 图纸在线查看与确认
- [ ] 点击行任意位置可进入图纸在线查看页
- [ ] 图纸文件正常加载，支持缩放与平移
- [ ] 未确认时显示 [Confirm Reading] 按钮；确认后显示 `✓ Confirmed on {date}`
- [ ] 确认记录 `deviceInfo` 正确记录为 `PC`
- [ ] 确认后返回列表页，Status 列更新为 `✓ Read`

### Markups Tab
- [ ] Markups Tab 仅显示 `ACTIVE` 状态的局部更新
- [ ] 局部更新按发布时间降序排列
- [ ] 未确认条目显示 `[New]` 标签和 `[Mark as Read]` 按钮
- [ ] 点击 `[Mark as Read]` 成功后按钮替换为 `✓ Read`，刷新后状态持久保留
- [ ] 已确认状态与 APP 端同步（在 APP 端确认后，PC 端刷新也显示 `✓ Read`）
- [ ] 点击 `●{n}` 角标直接激活 Markups Tab

### PC 端站内通知
- [ ] 局部更新发布后，已分配该图纸的 Site Engineer 在 PC 顶部铃铛处收到通知
- [ ] 未被分配该图纸的用户不收到通知
- [ ] 点击通知正确跳转到对应图纸页并自动激活 Markups Tab
- [ ] 点击后通知标记已读，角标数量正确减少
- [ ] "Mark all as read" 功能正常生效
