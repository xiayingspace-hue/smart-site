# Smart Site 多端产品需求管理规范

## 产品端说明

Smart Site 系统包含三个客户端，共享同一套后端服务和业务逻辑：

| 端 | 目录标识 | 技术栈 | 目标用户 | 说明 |
|----|---------|--------|----------|------|
| APP 移动端 | `app/` | UNIAPP + Vue 2 | 工地管理人员（现场） | iOS & Android 原生 APP |
| H5 移动端 | `h5/` | 待定 | 工地工人、分包商 | 浏览器/微信内嵌 H5 页面 |
| PC 管理端 | `pc/` | Vue 2 + Element UI | 系统管理员、项目经理 | 桌面浏览器 Web 管理后台 |

## 文件命名规范

### 需求文档

```
REQ-{编号}-{端标识}.md

示例：
  REQ-001-shared.md    → 登录功能的共享业务逻辑、API 定义
  REQ-001-app.md       → APP 端登录和首页的 UI 布局、交互、样式
  REQ-001-h5.md        → H5 端登录和首页的 UI 布局、交互、样式
  REQ-001-pc.md        → PC 端登录和管理后台首页
```

### 输出文档

```
{角色}-REQ-{编号}-{端标识}.md

示例：
  UI-DESIGN-SYSTEM.md         → 跨端设计系统（共享设计令牌）
  UI-REQ-001-app.md           → APP 端 UI 设计规范
  UI-REQ-001-h5.md            → H5 端 UI 设计规范
  UI-REQ-001-pc.md            → PC 端 UI 设计规范
  FE-REQ-001-app.md           → APP 端前端开发说明
  BE-REQ-001.md               → 后端开发说明（不区分端）
  QA-REQ-001-app.md           → APP 端测试用例
```

## 需求文档拆分原则

### shared（跨端共享）放什么？
- 业务规则和业务流程定义
- API 接口定义（端点、请求/响应格式、错误码）
- 数据模型和字段定义
- 权限和角色定义
- 会话管理逻辑（Token、租户上下文）
- 验证规则（字段校验逻辑，不含 UI 表现）

### 各端（app / h5 / pc）放什么？
- 页面布局和结构（各端不同）
- 视觉样式和设计令牌（各端不同）
- 交互方式和手势（各端不同）
- 端特有功能（如 APP 的扫码登录、H5 的微信授权、PC 的批量操作）
- 端特有的技术说明（技术栈、打包方式等）
- 端特有的非功能需求（屏幕适配、兼容性等）
- 引用 shared 文档中的业务规则和 API 定义

### 需求文档引用方式

各端需求文档通过引用 shared 文档来保持业务一致性：

```markdown
## 业务规则

> 详见 [REQ-001-shared.md](../shared/REQ-001-shared.md) — 第 3 节「登录业务规则」

## API 接口

> 详见 [REQ-001-shared.md](../shared/REQ-001-shared.md) — 第 4 节「API 接口定义」
```
