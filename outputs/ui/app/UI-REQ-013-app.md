# UI-REQ-013-app.md

## SE“我的任务”与消息通知页面 UI 说明（APP端）

### 1. 首页Progress Management模块
- 首页主要区域新增“Progress Management”模块，图标及名称与PC端保持一致
- SE登录后可在该模块下看到“我的任务”入口，点击进入SE专属任务列表页面
- 支持首页模块红点/数字提醒，提示有新任务或未读消息

### 2. 消息通知中心与入口聚合
- APP首页底部有notification（通知）入口，点击后进入通知中心页面
- 通知中心顶部显示两个Tab：
  1. Todo（待办）：显示所有与当前用户相关的待办事项（如新任务、待反馈、待确认等），支持一键跳转到对应任务或操作页面
  2. 消息通知：显示所有系统消息、任务变更、进度反馈、问题处理等通知，支持已读/未读状态切换
- 用户可在Tab间切换，快速区分待办与消息
- “我的任务”页面可通过首页Progress Management模块或Todo Tab进入

### 3. 我的任务页面（SE专用）
- 任务列表：卡片/列表样式，字段包括任务名称、计划时间、状态、优先级、操作按钮等
- 顶部为筛选/搜索区（状态、时间、优先级等）
- 新任务高亮显示，支持一键筛选
- 任务详情页：
  - 展示任务基本信息、工序要求、计划/实际时间、优先级等
  - 进度填报入口（如“开始施工”“完成施工”“异常上报”等按钮）
  - 工单反馈、附件上传、问题上报等操作
  - 问题汇报tab展示所有历史问题及处理反馈

### 4. 新建/反馈/上报交互
- 进度填报、工单反馈、问题上报均为弹窗/底部弹出表单，字段简洁、移动端友好
- 支持拍照上传、语音输入等移动端特性

### 5. 消息通知交互
- 新任务分配、任务变更、作废、进度反馈、问题处理等场景均有消息通知
- 通知内容格式与需求文档一致
- 消息中心统一查看，未读高亮，通知可直达任务详情/问题tab

### 6. 权限与入口说明
- “我的任务”入口仅SE可见，CM/PM不可见
- SE仅能操作自己被分配的任务
- 入口聚合：不再单独出现在底部导航栏或侧边栏，统一归入首页模块和消息通知中心
