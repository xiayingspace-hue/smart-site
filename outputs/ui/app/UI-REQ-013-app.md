# UI-REQ-013-app.md

## SE"My Task"APP 端 UI 说明

> 对应需求：REQ-013
> 平台：APP 端（UNIAPP + Vue2）
> 影响页面：My Task 列表页、任务详情页（工序任务详情）

---

### 1. 设计目标

- 为 SE 提供清晰的任务列表与工序任务操作页面
- 支持三态工作流：Not Started → In Progress → Completed（不可逆）
- Start Task 无需二次确认，直接更新状态；Mark as Complete 需二次确认弹框
- 实际开始/结束时间由系统自动记录，SE 只读
- 问题上报入口始终保持可用（任务非已作废状态）

---

### 2. 页面清单

| 页面 | 类型 | 说明 |
|------|------|------|
| My Task 列表页 | 全页面 | 顶部 Tab 筛选 + 任务卡片列表 |
| 工序任务详情页 | 全页面 | 工序基本信息 + 操作区 |

---

#### 3.3 操作区按钮规则

| 按钮 | Not Started | In Progress | Completed |
|------|-------------|-------------|-----------|
| Start Task | ✅ 可点击（主色按钮） | ⛔ disabled | ⛔ disabled |
| Mark as Complete | ⛔ disabled | ✅ 可点击（主色按钮） | ⛔ disabled |
| Report Issue | ✅ 可点击 | ✅ 可点击 | ✅ 可点击 |

**按钮样式：**
- 可点击的主操作按钮（Start Task / Mark as Complete）：实色填充，主题蓝 `#1890FF`
- disabled 按钮：背景 `#CCCCCC`，文字 `#FFFFFF`，不可触发点击事件
- Report Issue：边框按钮（Outline 样式）

---

#### 3.4 Start Task 交互

1. SE 点击 **Start Task** 按钮
2. **无确认弹框**，直接发起请求
3. 请求成功后：
   - 状态标签切换为 **In Progress**（橙色）
   - 实际开始时间字段更新为当前时间
   - Start Task 按钮变为 disabled
   - Mark as Complete 按钮变为可点击

---

#### 3.5 Mark as Complete 交互

1. SE 点击 **Mark as Complete** 按钮
2. 弹出确认弹框：

```
┌────────────────────────────────┐
│  确认完成                       │
│  确认将此工序任务标记为完成？   │
│  完成后将无法修改。              │
│            [取消]  [确认]       │
└────────────────────────────────┘
```

3. 点击"确认"后：
   - 状态标签切换为 **Completed**（绿色）
   - 实际结束时间字段更新为当前时间
   - Start Task 和 Mark as Complete 均变为 disabled
   - 基本信息区域整体变为只读（不可编辑）

---

```
┌─────────────────────────────────┐
```

---

### 4. 设计约束

- APP 端使用 UNIAPP 组件体系，状态标签用 `<u-tag>` 或自定义 tag 组件
- 详情页基本信息区域使用纵向字段列表，字段名左对齐，字段值右对齐
- 操作区固定在页面底部，避免被键盘遮挡
- disabled 按钮应从视觉和交互层面同时阻止操作（opacity + pointer-events）
- 实际开始/结束时间字段始终展示（无数据时显示"—"，不隐藏）

---

### 5. 验收条件

- [ ] 列表页顶部 Tab 支持 All / Not Started / In Progress / Completed 四类筛选
- [ ] 卡片状态标签展示三态颜色（灰/橙/绿）
- [ ] 详情页展示实际开始/结束时间（系统记录，只读，未记录时显示"—"）
- [ ] Start Task 按钮：仅 Not Started 状态可点，点击后无弹框直接变 In Progress
- [ ] Mark as Complete 按钮：仅 In Progress 状态可点，点击后有确认弹框
- [ ] Completed 状态：两个主操作按钮均置灰，信息区只读
- [ ] Report Issue 在所有状态下可点击
