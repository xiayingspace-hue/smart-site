# UI-REQ-013-app.md

## SE"My Task"APP 端 UI 说明

> 对应需求：REQ-013
> 平台：APP 端（UNIAPP + Vue2）
> 影响页面：首页 Progress Management 模块、My Task 工序任务列表页、工序任务详情页、标记完成确认弹框、问题上报表单

---

### 1. 设计目标

- 为 SE 在移动端提供专属工序任务查看与执行入口，首页一键直达
- 任务卡片简洁展示关键信息，支持状态筛选快速定位未完成任务
- 标记完成操作需二次确认，弹框明确告知锁定后果
- 问题上报入口始终可用，不受任务状态影响

---

### 2. 页面清单

| 页面 / 组件 | 类型 | 说明 |
|-------------|------|------|
| 首页 Progress Management 模块 | 首页模块 | 角色动态显示入口，SE 显示 My Task |
| My Task 工序任务列表页 | 全页面 | 筛选区 + 任务卡片列表 |
| 工序任务详情页 | 全页面 | 基本信息（只读）+ 操作区 |
| 标记完成确认弹框 | ActionSheet / Dialog | 点击"标记完成"后弹出 |
| 问题上报表单页 | 全页面 / 半屏弹框 | 点击"问题上报"进入 |

---

### 3. 页面详细说明

---

#### 3.1 首页 Progress Management 模块

```
┌────────────────────────────┐
│  Progress Management       │
│  ┌─────────┐  ┌─────────┐  │
│  │ My Task │  │  ...    │  │
│  │  [图标] │  │         │  │
│  │ 3 未读  │  │         │  │
│  └─────────┘  └─────────┘  │
└────────────────────────────┘
```

**交互说明：**
- SE 角色：仅显示 **My Task** 入口
- CM 角色：显示 CM 专属入口（REQ-012）
- PM 角色：显示 PM 专属入口（REQ-011）
- My Task 图标右上角显示数字角标（未读消息数 / 未完成任务数），无未读时隐藏角标

---

#### 3.2 My Task 工序任务列表页

```
┌────────────────────────────────┐
│  < 返回    My Task             │
├────────────────────────────────┤
│  [全部 ▼] [未完成] [已完成]    │
│  [优先级筛选 ▼] [时间筛选 ▼]  │
├────────────────────────────────┤
│  ┌──────────────────────────┐  │
│  │ 土方开挖                  │  │
│  │ 上级任务：ACT-001 xxxxxx │  │
│  │ 2025-06-01 ~ 2025-06-10  │  │
│  │ 优先级：High    [未完成] │  │
│  └──────────────────────────┘  │
│  ┌──────────────────────────┐  │
│  │ 基础浇筑                  │  │
│  │ 上级任务：ACT-001 xxxxxx │  │
│  │ 2025-06-05 ~ 2025-06-15  │  │
│  │ 优先级：Medium  [已完成] │  │
│  └──────────────────────────┘  │
└────────────────────────────────┘
```

**卡片字段：**

| 字段 | 说明 |
|------|------|
| 工序任务名称 | 加粗显示，卡片标题 |
| 所属上级任务 | 灰色小字，"上级：ACT-xxx 任务名" |
| 计划开始/结束时间 | 日期格式 YYYY-MM-DD |
| 优先级 | 彩色标签（High 红 / Medium 橙 / Low 绿） |
| 完成状态 | 右侧标签：未完成（灰）/ 已完成（绿） |

**筛选区：**
- 状态快速切换：全部 / 未完成 / 已完成（Tab 形式）
- 优先级下拉筛选
- 计划时间范围筛选
- 支持下拉刷新

---

#### 3.3 工序任务详情页

```
┌────────────────────────────────┐
│  < 返回    土方开挖             │
├────────────────────────────────┤
│  基本信息                      │
│  ─────────────────────────     │
│  工序名称：土方开挖             │
│  上级任务：ACT-001 xxxxxx      │
│  WBS：1.2.3                    │
│  计划时间：06-01 ~ 06-10       │
│  优先级：High                  │
│  备注：—                       │
│  完成状态：未完成               │
├────────────────────────────────┤
│  操作区                        │
│  [标记完成]    [问题上报]       │
└────────────────────────────────┘
```

**基本信息（只读）：**

| 字段 | 说明 |
|------|------|
| 工序任务名称 | 文本 |
| 所属上级任务 | 文本 |
| WBS | 文本，无数据显示"—" |
| 计划开始/结束时间 | 文本 |
| 优先级 | 彩色标签 |
| 备注 | 文本，无数据显示"—" |
| 当前完成状态 | 未完成（灰）/ 已完成（绿，带锁定图标） |

**操作区：**
- **标记完成**按钮：
  - 未完成状态：蓝色主按钮，可点击
  - 已完成状态：置灰 disabled，显示"已完成"
- **问题上报**按钮：白色次要按钮，任意状态均可点击

---

#### 3.4 标记完成确认弹框

```
┌────────────────────────────────┐
│                                │
│  确认标记完成？                 │
│                                │
│  标记完成后将锁定该工序，       │
│  如需修改请联系 CM，            │
│  确认继续？                    │
│                                │
│  [取消]         [确认完成]     │
│                                │
└────────────────────────────────┘
```

**交互说明：**
- 点击"标记完成"后弹出此确认弹框
- **取消**：关闭弹框，工序状态不变
- **确认完成**：
  - 工序任务状态变更为"已完成"并锁定
  - 标记完成按钮置灰
  - 系统自动重新计算上级任务进度
  - 显示成功 Toast："已标记完成"
- 弹框使用 uni.showModal 或自定义 Dialog，背景半透明遮罩

---

#### 3.5 问题上报表单

```
┌────────────────────────────────┐
│  < 返回    问题上报             │
├────────────────────────────────┤
│  关联工序：土方开挖（只读）    │
│  问题类型*：[下拉选择 ▼]       │
│  问题描述*：                   │
│  ┌──────────────────────────┐  │
│  │ 请输入问题描述...         │  │
│  └──────────────────────────┘  │
│  影响范围：[文本输入]           │
│  建议方案：[文本输入]           │
│  附件：[+ 上传图片/文件]（可选）│
├────────────────────────────────┤
│           [提交上报]            │
└────────────────────────────────┘
```

**字段说明：**

| 字段 | 必填 | 说明 |
|------|------|------|
| 关联工序 | — | 只读，自动带出当前工序名称 |
| 问题类型 | ✅ | 下拉选择 |
| 问题描述 | ✅ | 多行文本，最大 500 字 |
| 影响范围 | 可选 | 文本输入 |
| 建议方案 | 可选 | 文本输入 |
| 附件 | 可选 | 图片/文件，最多 5 个 |

- 提交后显示成功 Toast："上报成功，已通知 CM"
- 提交后返回详情页

---

### 4. 设计约束

- 整体风格遵循 APP 端设计规范（UNIAPP），与现有 APP 页面一致
- 卡片圆角 8px，背景白色，阴影 `0 2px 8px rgba(0,0,0,0.08)`
- 未完成状态标签：灰底 `#F5F5F5`，文字 `#999`
- 已完成状态标签：绿底 `#E8F5E9`，文字 `#4CAF50`
- 已完成工序在详情页操作区标记完成按钮背景 `#CCCCCC`，不可点击
- 确认弹框按钮："取消"用灰色文字，"确认完成"用蓝色/品牌色，避免误操作

---

### 5. 验收条件

- [ ] SE 登录后首页 Progress Management 模块显示 My Task 入口，CM/PM 不显示
- [ ] My Task 入口有角标提示未完成/未读数量
- [ ] My Task 列表页卡片展示工序名称、上级任务、时间、优先级、完成状态
- [ ] 支持按完成状态快速筛选（全部 / 未完成 / 已完成）
- [ ] 点击卡片进入详情页，所有字段只读
- [ ] 未完成工序"标记完成"按钮可点击，已完成置灰
- [ ] 点击"标记完成"弹出确认弹框，确认后状态锁定，Toast 提示成功
- [ ] 问题上报按钮任意状态可点击，表单字段完整，提交后返回详情页
