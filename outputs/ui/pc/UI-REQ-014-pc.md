# UI-REQ-014-pc.md

## 图纸管理用户反馈迭代 UI 说明（PC 端）

> 对应需求：REQ-014（REQ-003/004/005 第一版用户反馈汇总）
> 平台：PC 端（Vue2 + Element UI）
> 影响页面：工程图纸管理页、图纸版本详情弹框、Markup 历史弹框

---

### 1. 设计目标

| 反馈编号 | 涉及需求 | 改动摘要 |
|----------|----------|----------|
| FB-001 | REQ-003 | 上传新版本时，Drawing Code 字段支持手动修改 |
| FB-002 | REQ-003 | 图纸版本详情弹框新增 Attachments 附件区域 |
| FB-003 | REQ-004 | 操作列移除"+Markup"按钮，入口移入 Markup 历史弹框 |
| FB-004 | REQ-004 | Markup 历史列表新增"图纸编号（Drawing Code）"列 |
| FB-005 | REQ-004 / REQ-005 | SE 角色 PC 端完全隐藏 Markup 相关入口 |
| FB-006 | REQ-003 / REQ-005 | SE 视角只显示 Drawing Code，隐藏系统版本号（Ver）；非SE角色两者均显示 |

---

### 2. 改动点说明

---

#### 2.1 FB-001 上传新版本时支持修改 Drawing Code

**涉及位置：**
- 图纸列表页 → 上传新版本弹框（Drawing Code 字段）
- 图纸版本列表（历史版本 Drawing Code 只读展示）

**交互说明：**
- 上传新版本弹框中，`Drawing Code` 字段默认回填当前图纸的编号，允许用户手动修改
- 修改后的值仅对本次新版本生效，历史版本的 Drawing Code 不受影响
- 输入框失焦或点击提交时，校验新 Drawing Code 在当前项目下的唯一性；若与其他图纸编号冲突，显示提示：`"该 Drawing Code 已存在，请重新输入"`，提交按钮禁用
- 历史版本的 Drawing Code 在版本列表中只读展示，不可编辑

**上传新版本弹框字段（建议）：**

| 字段 | 是否可编辑 | 说明 |
|------|-----------|------|
| Drawing Code | ✅ 可编辑 | 默认回填当前值，可修改，需唯一性校验 |
| 上传 PDF | ✅ 必填 | 选择新版本文件 |
| 备注 | ✅ 可选 | 版本说明 |

**版本列表字段顺序（建议）：**

| 字段 | 说明 |
|------|------|
| Drawing Code | 该版本对应的图纸编号，历史版本只读 |
| Ver | 系统自动递增版本序号 |
| 上传时间 | 版本 PDF 上传时间 |
| 上传人 | 上传操作人 |
| 文件大小 | PDF 文件大小 |
| 状态 | 审批状态 |
| 操作 | 查看、下载等 |

---

#### 2.2 FB-002 图纸版本附件（Attachments）区域

**涉及位置：**
- 图纸版本详情弹框（新增区域，位于 PDF 主文件信息下方）
- 上传新版本弹框（可选，支持同步上传附件）

**版本详情弹框结构示意：**

```
┌─────────────────────────────────────────────────────────────┐
│  版本详情                                          [关闭]   │
├─────────────────────────────────────────────────────────────┤
│  Drawing Code：DWG-001    Ver：1    上传时间：2026-04-01    │
│  PDF 主文件：[xxx.pdf]    [查看]  [下载]                    │
│  审批状态：已审批                                           │
├─────────────────────────────────────────────────────────────┤
│  附件（Attachments）                     [+ 上传附件]       │
│  ┌──────────┬──────┬───────┬────────────┬──────┬─────────┐  │
│  │ 文件名   │ 类型 │ 大小  │  上传时间  │ 备注 │  操作   │  │
│  ├──────────┼──────┼───────┼────────────┼──────┼─────────┤  │
│  │ xxx.dwg  │ CAD  │ 2 MB  │ 2026-04-01 │  -   │下载 删除│  │
│  │ xxx.docx │ 文档 │ 0.5MB │ 2026-04-01 │ 说明 │下载 删除│  │
│  └──────────┴──────┴───────┴────────────┴──────┴─────────┘  │
└─────────────────────────────────────────────────────────────┘
```

**交互说明：**
- 附件区域仅在版本详情弹框中展示，不在图纸列表主表格中显示
- "上传附件"按钮：点击后弹出文件选择器，支持多文件同时选择，不限文件类型
- 每个附件上传时可填写备注（可选），用于说明文件用途
- 附件列表字段：文件名、类型（取文件后缀）、大小、上传时间、备注、操作（下载 / 删除）
- 删除操作需二次确认弹窗：`"确认删除该附件？"`，确认后删除
- 附件与具体版本绑定，不同版本的附件独立管理
- 附件区域不影响 PDF 主文件的审批、QR 码、SE 分配等主流程

**权限控制：**

| 操作 | 有权限的角色 |
|------|-------------|
| 上传附件 | PDF 上传人 / 有图纸管理权限的角色 |
| 删除附件 | PDF 上传人 / 有图纸管理权限的角色（显示删除按钮） |
| 下载附件 | 所有有图纸访问权限的角色（含 SE，与 PDF 下载权限一致） |
| 查看附件列表 | 所有有图纸访问权限的角色（SE 可见列表，但不显示删除按钮） |

#### 2.3 FB-003 精简操作列，"+Markup"入口移入历史弹框

**涉及位置：**
- 图纸列表页 → 操作列
- Markup 历史记录弹框

**操作列改动对比：**

| 操作按钮 | 改动前 | 改动后 |
|----------|--------|--------|
| 查看 PDF | ✅ 保留 | ✅ 保留 |
| 审批 | ✅ 保留 | ✅ 保留 |
| + Markup | ✅ 显示在操作列 | ❌ 从操作列移除 |
| Markup 历史（图标） | ✅ 保留 | ✅ 保留 |

**Markup 历史弹框改动：**

```
┌──────────────────────────────────────────────────────────────┐
│  Markup 历史记录                            [+ Add Markup]   │
├──────┬──────────────┬─────────────┬────────────┬────────────┤
│  #   │   文件名     │ Drawing Code│  上传时间  │    操作    │
├──────┼──────────────┼─────────────┼────────────┼────────────┤
│  1   │ markup1.pdf  │  DWG-001    │ 2026-03-10 │ 查看 合并  │
│  2   │ markup2.pdf  │  DWG-002    │ 2026-02-20 │ 查看 合并  │
└──────┴──────────────┴─────────────┴────────────┴────────────┘
```

**交互说明：**
- 操作列"Markup 历史"图标按钮保留，点击后弹出 Markup 历史弹框
- 弹框右上角新增"+ Add Markup"主操作按钮，点击进入 Markup 上传流程（功能与原"+Markup"一致）
- 操作列按钮数量减少，视觉更简洁

---

#### 2.4 FB-004 Markup 历史列表新增"图纸编号（Drawing Code）"列

**涉及位置：**
- Markup 历史记录弹框 → 列表新增列

**字段说明：**

| 属性 | 说明 |
|------|------|
| 字段名称 | `图纸编号`（英文：`Drawing Code`） |
| 展示值 | Markup 创建时对应图纸的 Drawing Code，如 `DWG-001`、`S-A01-001` |
| 数据来源 | 系统自动记录，Markup 创建时自动关联当前图纸的 Drawing Code，无需用户操作 |
| 是否可编辑 | ❌ 只读，创建后不可修改 |
| 列位置 | "文件名"列之后（参见 2.3 弹框示意图） |

---

#### 2.5 FB-005 SE 角色完全隐藏 Markup 相关入口（PC 端）

**涉及位置：**
- 图纸列表页 → 操作列（SE 视角）
- Markup 历史弹框（SE 无法触达）

**权限规则：**

| UI 元素 | 上传人 / PM / CM | SE角色 |
|---------|------------------|--------|
| 操作列"Markup 历史"图标按钮 | ✅ 显示 | ❌ 隐藏 |
| Markup 历史弹框 | ✅ 可访问 | ❌ 不可访问 |
| Markup 数量角标（如有） | ✅ 显示 | ❌ 隐藏 |
| 弹框内"+ Add Markup"按钮 | ✅ 显示（有权限者） | ❌ 不可访问 |

**说明：**
- SE 视角下操作列不渲染任何 Markup 相关元素，布局不留空白占位
- 后端接口层同样需做权限拦截，UI 隐藏为第一道防线

---

#### 2.6 FB-006 SE 视角隐藏系统版本号（Ver），仅显示 Drawing Code

**涉及位置：**
- 图纸列表页（版本相关列）
- 图纸版本详情弹框（版本信息区）

**各角色字段展示差异：**

| 字段 | 上传人 / PM / CM | SE |
|------|------------------|----|
| Drawing Code | ✅ 显示 | ✅ 显示 |
| 系统版本号（Ver） | ✅ 显示 | ❌ 隐藏 |

**图纸版本详情弹框 - 版本信息区对比：**

非SE角色视角：
```
Drawing Code：DWG-001
系统版本（Ver）：2
上传时间：2026-04-01
```

SE角色视角：
```
Drawing Code：DWG-001
上传时间：2026-04-01
```

**交互说明：**
- SE 视角中系统 Ver 字段完全不渲染，不留空白占位，布局保持紧凑
- 图纸列表中若有版本号列，SE 视角该列只展示 Drawing Code，不显示 Ver 编号

---

### 3. 各角色 PC 端视角差异总结

| 功能 | 上传人 / PM / CM | SE |
|------|------------------|----|
| 上传新版本时修改 Drawing Code | ✅ | ❌（只读展示） |
| 附件上传 / 删除 | ✅（有权限者） | ❌ |
| 附件下载 | ✅ | ✅（已分配版本） |
| 附件列表查看 | ✅ | ✅（无删除按钮） |
| Markup 历史入口 | ✅ | ❌ 隐藏 |
| + Add Markup | ✅（有权限者） | ❌ 隐藏 |
| Markup 历史列表"图纸编号（Drawing Code）"列 | ✅ 可见 | ❌ 不可访问 |
| 系统版本号（Ver）显示 | ✅ | ❌ 隐藏 |

### 4. 验收条件

- [ ] 上传新版本弹框中 `Drawing Code` 字段可编辑，默认回填当前值，重复值有实时校验提示，历史版本 Drawing Code 只读
- [ ] 版本详情弹框中展示 Attachments 区域，支持多文件上传/下载/删除，权限符合角色规则，SE 无删除按钮
- [ ] 图纸列表操作列中"+Markup"按钮已移除，Markup 历史弹框右上角出现"+ Add Markup"按钮
- [ ] Markup 历史列表新增"图纸编号（Drawing Code）"列，值由系统自动记录，只读
- [ ] SE 角色 PC 端图纸列表操作列中不出现任何 Markup 相关入口，隐藏后布局无空白占位
- [ ] SE 角色 PC 端图纸列表及版本详情中不显示系统 Ver 字段，仅展示 Drawing Code；非SE角色两者均显示，布局无空白占位
