

# UI说明文档 — REQ-008 公司层级工序模板配置（新版布局）

## 1. 页面入口
- 视角：Company Insights（公司层级）
- 左侧导航栏：Project Progress
- Project Progress 下有两个二级菜单：
	- Dictionary Management
	- Process Settings

## 2. 页面结构

### 2.1 Project Progress 二级菜单交互
- 点击“Dictionary Management”后，主内容区显示空白页面（该功能已实现，无需重复开发）。
- 点击“Process Settings”后，主内容区显示公司层级构件工序模板配置页面。

### 2.2 Process Settings 页面结构
- 顶部为专业Tab栏（如：Structural、Architectural、M&E等），点击切换不同专业
- 专业Tab由公司字典数据生成，不支持新增、编辑、删除
- 左侧为构件类型列表，展示当前专业下所有构件类型（如：Pile Test、RC Pile等），由字典数据生成，不支持新增、编辑、删除
- 右侧为工序表格，展示当前构件类型下所有工序
	- 表格字段建议包含：序号、工序名称、权重、标准工期、可选/必选、操作
	- 仅工序支持新增、编辑、删除、批量添加、排序

### 2.3 其他说明
- 顶部Tab、左侧构件、右侧工序表格均为联动，切换专业或构件时右侧表格自动刷新
- 支持工序的批量添加和权重输入校验（如总和需为100%或提示）

## 3. 主要交互
- 仅工序支持新增、编辑、删除、禁用、批量添加、排序
- 专业和构件类型均来源于公司字典，不支持用户自定义
- 删除/禁用工序交互：
	- 未被任何项目引用的工序可直接删除
	- 已被项目引用或有历史数据的工序，“删除”按钮置灰，仅可“禁用”
	- 禁用后工序在新项目/新任务中不可再选用，历史项目和数据不受影响
	- 删除/禁用操作前弹窗提示，需二次确认
	- 所有操作均需后端校验

## 4. 设计要点
- 保持与现有管理后台风格一致（Element UI风格，主色#4A90D9）
- 顶部Tab、左侧列表、右侧表格布局清晰，间距合理
- 操作按钮悬浮显示，避免界面拥挤
- 表单校验清晰，错误提示明显

## 5. 空态与反馈
- 无工序时表格显示空态提示和“新增工序”按钮
- 操作成功/失败有Toast提示

## 6. 响应式适配
- 建议仅支持PC端大屏，1280px及以上
