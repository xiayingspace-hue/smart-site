## PM“我的任务”菜单与任务执行页面 UI 说明

### 1. 页面入口与菜单
- 菜单名称：我的任务（PM）
- 入口位置：左侧 Progress Management 菜单下，仅 PM 角色可见

### 2. 页面整体结构
```
┌──────────────────────────────────────────────────────────────┐
│  顶部操作区：筛选/搜索区   [新建任务] [批量操作] ...         │
├──────────────────────────────────────────────────────────────┤
│  任务表格（字段示例）：                                      │
│  ┌─────────┬─────────────┬─────┬────────────┬─────┬─────┬─────┐│
│  │ 任务ID  │ 任务名称    │WBS  │ 计划时间   │状态 │CM   │操作 ││
│  ├─────────┼─────────────┼─────┼────────────┼─────┼─────┼─────┤│
│  │ ...     │ ...         │...  │ ...        │...  │...  │...  ││
│  └─────────┴─────────────┴─────┴────────────┴─────┴─────┴─────┘│
├──────────────────────────────────────────────────────────────┤
│  任务详情页（弹窗/侧滑）：                                   │
│  ┌────────────────────────────────────────────────────────┐ │

└──────────────────────────────────────────────────────────────┘
```
- 具体UI以设计稿为准

### 3. 主要交互与功能
#### 3.1 顶部操作区
- 筛选/搜索：支持按任务状态、时间、WBS等筛选
- 新建任务：按钮位于右上角，点击后从右侧以侧滑弹框方式进入任务创建
- 批量操作：如批量分配、批量作废等（按权限显示）

#### 3.2 任务表格
- 字段顺序、时间格式、样式等与Planning端Master Program页面保持一致
- 字段建议：Activity ID、Activity Name、WBS、计划开始/结束时间、实际开始/结束时间、Deviation、Status、Assigned CM、操作
- 状态字段高亮，标签颜色与状态说明一致
- 新任务高亮显示，顶部显示新任务数量，支持一键筛选
- 操作列包含：分配CM、汇报问题等按钮（按权限显示/置灰）

#### 3.3 新建任务弹框
- 右侧侧滑弹框，样式与Planning端一致
- 字段：关联主计划任务（下拉/搜索）、任务名称、WBS、计划开始/结束时间、优先级、可选分配CM、备注等，支持批量新建

#### 3.4 问题汇报入口
- 操作列每行有“汇报问题”按钮（图标或文字），点击弹出问题汇报弹窗
- 任务详情页也有“汇报问题”按钮
- 问题汇报弹窗字段：问题类型、描述、影响、建议、附件等

#### 3.5 任务详情页
- 包含任务基本信息、下级任务列表、问题汇报tab、操作按钮等
- 问题汇报tab展示所有历史问题及处理反馈

#### 3.6 消息通知
- 新任务分配、任务变更、作废、进度反馈、问题汇报、逾期预警、退回、资料补充、审批、评论、完成确认、分解合并等场景均有消息通知
- 通知内容格式详见需求文档
- 消息中心统一查看，未读高亮，通知可直达任务详情/问题tab

#### 3.7 状态与权限
- 任务状态标签、颜色、切换规则与需求文档一致
- 不同角色仅能看到与自身相关的菜单和页面

### 4. 角色区分与入口
- PM菜单为“我的任务（PM）”，仅PM可见
- CM菜单为“我的任务（CM）”或“下级任务”，仅CM可见
- 页面标题、面包屑等处有角色标识
