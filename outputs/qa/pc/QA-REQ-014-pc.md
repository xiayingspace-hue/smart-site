# QA 测试说明文档 — REQ-014 图纸管理用户反馈迭代（PC 端）

> **来源需求**: `requirements/pc/REQ-014-pc.md`
> **产品**: SMART SITE SYSTEM
> **平台**: PC 管理端
> **测试用例编号格式**: `TC-014-PC-XX`
> **生成日期**: 2026-04-28

---

## 1. 测试目标

覆盖 REQ-014 PC 端 6 条用户反馈（FB-001 ～ FB-006）的功能、权限、边界场景，确保：
- Drawing Code 可编辑且唯一性校验正确
- 附件 CRUD 及权限符合预期
- Markup 入口位置调整后行为正常
- SE 角色无法访问 Markup，无法看到系统版本号 Ver

---

| 反馈编号 | 测试模块 | 用例数 |
|----------|----------|--------|

| FB-005 | SE 角色 Markup 隐藏 | 4 |

## 3. 测试场景与用例

---

### 3.1 FB-001 Drawing Code 修改与唯一性校验

| 用例 ID | 前置条件 | 操作步骤 | 预期结果 |
|---------|----------|----------|----------|
| TC-014-PC-01 | 以 PM 角色登录；存在图纸 A（Drawing Code: DWG-001） | 对图纸 A 点击"上传新版本"，修改 Drawing Code 为 `DWG-002`（项目内未使用），提交 | 新版本上传成功；Drawing Code 更新为 DWG-002；历史版本 Drawing Code 仍显示 DWG-001 |
| TC-014-PC-02 | 同上；项目内已有图纸 B Drawing Code 为 DWG-003 | 上传新版本时，将 Drawing Code 修改为 `DWG-003` | 输入框失焦后显示错误提示"该 Drawing Code 已存在"；提交按钮不可点击 |
| TC-014-PC-03 | 同上 | 上传新版本弹框打开时，不修改 Drawing Code，直接提交 | 沿用原 Drawing Code，上传成功 |
| TC-014-PC-04 | 同上 | 将 Drawing Code 修改为空字符串后提交 | 显示"Drawing Code 不能为空"校验提示；提交失败 |
| TC-014-PC-05 | 同上 | 修改 Drawing Code 为仅含空格 | 校验失败（等同于空值或显示格式错误提示） |
| TC-014-PC-06 | 以 SE 角色登录（如 SE 有访问版本详情的权限，但无上传权限） | 查看上传新版本弹框（若可见） | Drawing Code 输入框为只读状态，不可编辑 |

| TC-014-PC-13 | 以 SE 角色登录；已被分配该图纸版本；有附件 | 点击附件 [下载] | 文件正常下载 |

## 4. 测试执行注意事项

- FB-001 唯一性校验需在同一项目下测试，跨项目相同 Drawing Code 不应报错（按需确认范围）
- FB-005 需测试所有可能入口（列表操作列、详情弹框、角标），并验证接口层 403
- FB-006 需覆盖列表页和详情弹框两个展示位置
- 边界场景：图纸无附件、无 Markup 时的空态显示均需覆盖
