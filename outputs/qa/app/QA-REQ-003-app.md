# QA 测试说明文档 — REQ-003 APP 端图纸查阅确认与审批

> **来源需求**: REQ-003-shared + REQ-003-app
> **平台**: APP 移动端（iOS & Android，UNIAPP）
> **生成日期**: 2026-04-08

---

## 1. 测试目标

验证 APP 端 Site Engineer 图纸列表显示、在线查看、查阅确认及审批 Todo 操作的正确性，以及 App Push 通知的接收与跳转。

---

## 2. 测试场景与用例

### 场景 1：图纸列表

| ID | 前置条件 | 测试步骤 | 预期结果 |
|----|---------|---------|---------|
| TC-003-APP-01 | SE 登录，有 ≥1 张已分配 ACTIVE 图纸 | 进入 Drawings 列表 | 仅显示分配给当前 SE 且 ACTIVE 的图纸 |
| TC-003-APP-02 | 未分配任何图纸 | 进入 Drawings 列表 | 显示空状态 "No drawings assigned to you yet." |
| TC-003-APP-03 | 有多张图纸 | 下拉刷新 | 列表刷新 |
| TC-003-APP-04 | 有 >20 张图纸 | 滚动到底部 | 自动加载更多，追加到列表末尾 |
| TC-003-APP-05 | 有图纸 | 在搜索框输入图纸编号关键词 | 列表过滤，仅显示匹配记录 |
| TC-003-APP-06 | 有图纸 | 在分类下拉框选择某分类 | 列表过滤为该分类 |

### 场景 2：图纸状态标识

| ID | 测试步骤 | 预期结果 |
|----|---------|---------|
| TC-003-APP-07 | 查看已确认图纸的卡片 | 右侧显示绿色 "✓ Read" |
| TC-003-APP-08 | 查看未确认图纸的卡片 | 右侧显示蓝色 "Confirm →" |
| TC-003-APP-09 | 有未读 Markup 的图纸 | 卡片右上角显示橙色圆点 |

### 场景 3：在线查看

| ID | 测试步骤 | 预期结果 |
|----|---------|---------|
| TC-003-APP-10 | 点击任意图纸进入详情 | 展示在线预览（PDF 或图片），页面顶部显示图纸编号和版本 |
| TC-003-APP-11 | PDF 预览后，使用双指捏合/展开 | 图纸缩放响应正确 |
| TC-003-APP-12 | 缩放后单指拖动 | 图纸平移正确 |

### 场景 4：Confirm Reading

| ID | 前置条件 | 测试步骤 | 预期结果 |
|----|---------|---------|---------|
| TC-003-APP-13 | 未确认图纸 | 在详情页底部点击 [✓ Confirm Reading] | 弹出确认对话框："Confirm that you have read this drawing?" |
| TC-003-APP-14 | 同上 | 确认对话框点击 [Confirm] | API 调用成功（deviceInfo=APP），底部按钮变为 "✓ Confirmed on {date}" |
| TC-003-APP-15 | 同上 | 确认后返回列表 | 该图纸 Status 变为 "✓ Read" |
| TC-003-APP-16 | 同上 | 确认对话框点击 [Cancel] | 无操作，按钮不变 |
| TC-003-APP-17 | 已确认图纸 | 查看详情 | 底部显示 "✓ Confirmed on {date}"，无操作按钮 |
| TC-003-APP-18 | 重复调用确认 API | 后端接口 | 返回 200，无报错（幂等）|

### 场景 5：审批 Todo

| ID | 前置条件 | 测试步骤 | 预期结果 |
|----|---------|---------|---------|
| TC-003-APP-19 | 审批人登录，有待审批图纸 | 进入 Todo 列表，找到 Drawing Approval 卡片 | 卡片显示图纸编号、名称、版本号、上传人、时间 |
| TC-003-APP-20 | 同上 | 点击 [View] | 跳转图纸在线预览页（无 Confirm Reading 按钮）|
| TC-003-APP-21 | 同上 | 点击 [Approve]，确认 | Toast 提示 "Drawing approved successfully"，Todo 卡片消失 |
| TC-003-APP-22 | 同上 | 点击 [Reject]，弹出底部 Sheet | Sheet 中含 "Comment (Required)" 文本框 |
| TC-003-APP-23 | 同上 | Comment 为空点击 [Confirm] | [Confirm] 按钮禁用 |
| TC-003-APP-24 | 同上 | 填写 Comment 后确认驳回 | Toast 提示 "Drawing rejected"，Todo 卡片消失 |
| TC-003-APP-25 | 同上 | 审批后刷新 Todo 列表 | 已处理的 Todo 不再显示 |

### 场景 6：App Push 通知

| ID | 前置条件 | 测试步骤 | 预期结果 |
|----|---------|---------|---------|
| TC-003-APP-26 | SE 已分配图纸，APP 在前台 | DC 触发图纸审批通过 | SE 收到 Toast 通知提示 |
| TC-003-APP-27 | SE 已分配图纸，APP 在后台 | DC 触发图纸审批通过 | SE 收到系统推送通知 |
| TC-003-APP-28 | 同上 | 点击推送通知 | APP 打开，跳转到图纸列表页 |
| TC-003-APP-29 | SE 未被分配该图纸 | DC 触发该图纸审批通过 | SE 不收到任何推送 |
| TC-003-APP-30 | APP 未启动（冷启动）| 点击推送通知启动 APP | APP 启动后跳转到图纸列表 |

## 4. 非功能测试

| 类型 | 测试点 |
|------|-------|
| 幂等 | Confirm Reading 重复调用无报错 |
| 安全 | SE 无法看到未分配给自己的图纸（直接 API 调用返回 403）|
| 兼容性 | iOS 15+ 和 Android 10+ 推送均可正常接收 |
| 推送权限 | 用户拒绝推送权限后，功能降级为轮询或 App 内 Badge |

---

## 5. 验收标准

- [ ] 图纸列表仅显示 ACTIVE + 已分配图纸
- [ ] Confirm Reading Happy Path 通过（含幂等）
- [ ] 审批通过/驳回 Happy Path 通过
- [ ] 驳回时 Comment 为空无法提交
- [ ] App Push 推送给正确 SE（不多发、不漏发）
- [ ] 冷启动推送跳转正常
