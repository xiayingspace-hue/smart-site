# QA 测试说明文档 — REQ-001 APP 端多租户登录与会话管理

> **来源需求**: REQ-001-shared + REQ-001-app
> **平台**: APP 移动端（iOS & Android，UNIAPP）
> **生成日期**: 2026-04-08

---

## 1. 测试目标

验证 APP 端多租户登录、Token 管理、权限加载及首页展示的正确性与安全性。

---

## 2. 测试场景与用例

### 场景 1：登录

| ID | 前置条件 | 测试步骤 | 预期结果 |
|----|---------|---------|---------|
| TC-001-APP-01 | 租户 `mcc` 存在，用户 `se_user1` 有效 | 输入 tenantCode=mcc，用户名和密码，点击 [Login] | 登录成功，跳转 APP 首页，底部 TabBar 可见 |
| TC-001-APP-02 | 同上 | 输入错误密码 | 提示 "Username or password incorrect" |
| TC-001-APP-03 | 同上 | tenantCode 填写不存在的租户 | 提示 "Tenant not found" |
| TC-001-APP-04 | 账号已禁用 | 输入该账号登录 | 提示 "Account is disabled" |
| TC-001-APP-05 | 网络断开 | 点击登录 | 提示网络错误，按钮恢复可点击状态 |
| TC-001-APP-06 | 分包商账号 | 在登录页输入分包商账号（通过 SUBCON 入口）| 登录成功，仅显示分包商可访问功能 |

### 场景 2：Token 管理

| ID | 测试步骤 | 预期结果 |
|----|---------|---------|
| TC-001-APP-07 | 登录后后台挂起超过 1h | 前台切回时发起 API 请求，自动 Refresh Token，用户无感知 |
| TC-001-APP-08 | 清除 App 本地数据（模拟 Refresh Token 丢失）| 发起请求时跳回登录页，提示 "Session expired" |
| TC-001-APP-09 | 登录后点击 [Logout] | 调用 logout API，清除本地存储，跳回登录页 |

### 场景 3：项目选择

| ID | 测试步骤 | 预期结果 |
|----|---------|---------|
| TC-001-APP-10 | 用户关联多项目，登录后 | 弹出项目选择页或 Picker，选择后进入首页 |
| TC-001-APP-11 | 用户仅关联一个项目 | 跳过选择直接进入首页 |

### 场景 4：首页功能入口

| ID | 前置条件 | 测试步骤 | 预期结果 |
|----|---------|---------|---------|
| TC-001-APP-12 | SE 角色登录 | 查看首页 | 显示 "Drawings" 和 "Todo" 入口 |
| TC-001-APP-13 | SUBCON 操作员角色登录 | 查看首页 | 显示 "Refueling" 入口，不显示 "Drawings" |

---

## 3. 界面验收

| ID | 检查项 | 预期结果 |
|----|--------|---------|
| UI-001-APP-01 | 登录按钮 | loading 状态期间禁用，防止重复提交 |
| UI-001-APP-02 | tenantCode 输入框 | APP 首页有专属 tenant 输入框（与 PC URL 不同）|
| UI-001-APP-03 | 底部 TabBar | 登录后显示（首页 / 图纸 / 我的等按角色配置）|

---

## 4. 非功能测试

| 类型 | 测试点 |
|------|-------|
| 兼容性 | iOS 15+ 和 Android 10+ 设备登录正常 |
| 安全 | 密码字段遮掩，键盘不显示密码建议 |
| 性能 | 登录响应 P99 < 500ms（网络正常时）|

---

## 5. 验收标准

- [ ] 正确账号可登录，首页正常渲染
- [ ] 错误提示内容与需求一致
- [ ] Token 自动刷新无感知
- [ ] 登出后本地数据完全清除
- [ ] iOS 和 Android 均通过兼容性测试
