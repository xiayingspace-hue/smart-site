# 项目背景说明

此目录用于产品经理记录项目的关键背景信息，帮助AI和团队成员理解整个产品的发展脉络。

## 背景信息结构

```
background/
├── README.md                   # 本说明文件
├── project-overview.md         # 项目概述
├── business-context.md         # 业务背景
├── market-analysis.md          # 市场分析
├── user-research.md            # 用户研究
└── key-decisions.md            # 关键决策记录
```

## 各文件说明

### project-overview.md
- 项目的总体介绍
- 项目目标和愿景
- 项目范围和边界
- 主要参与者和角色

### business-context.md
- 为什么启动这个项目
- 业务痛点和机会
- 预期的业务价值
- 成功指标

### market-analysis.md
- 市场现状分析
- 竞争对手分析
- 目标市场和用户
- 市场机会和威胁

### user-research.md
- 用户调研结果
- 用户画像和人设
- 用户痛点和需求
- 用户行为分析

### key-decisions.md
- 重要决策记录
- 决策的理由
- 决策的影响
- 后续跟进计划

## 如何使用

1. **产品经理** 在此目录中记录项目背景信息
2. **AI生成文档时** 可以参考这些背景信息，确保生成的文档与项目愿景一致
3. **团队成员** 可以通过这些文档快速了解项目背景
4. **新加入成员** 可以通过这些文档快速融入团队

## 更新频率

- 初期：详细记录所有背景信息
- 持续：当有重大决策或方向调整时，更新相应的背景文档
