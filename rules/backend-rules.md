# 后端开发转换规则

定义如何从需求文档生成后端开发说明文档。

## 转换规则

### 1. 业务逻辑提取
- 从需求文档的"功能需求"和"验收标准"中提取业务逻辑
- 识别所有需要实现的业务规则

### 2. 数据模型设计
- 从业务逻辑推导需要的数据模型
- 确定数据之间的关系

### 3. API设计
- 根据功能需求设计API接口
- 定义请求/响应格式

### 4. 性能和安全需求
- 从非功能需求提取性能要求
- 提取安全约束

## 示例转换

### 输入（需求文档）
```
需求：用户登录功能
功能需求：用户可以使用邮箱和密码登录，返回认证令牌
验收标准：登录成功返回token，失败返回错误信息
```

### 输出（后端开发说明）
```
业务逻辑：
- 验证邮箱格式
- 验证密码与存储的密码
- 生成认证令牌

API接口：
- POST /api/login
  输入: {email, password}
  输出: {token, user_info}
  
数据模型：
- User表

安全要求：
- 密码加密存储
- API速率限制
```
