# QA测试转换规则

定义如何从需求文档生成QA测试说明文档。

## 转换规则

### 1. 测试场景识别
- 从需求文档的"验收标准"推导测试场景
- 识别正常流程和异常流程

### 2. 测试用例生成
- 为每个验收标准生成对应的测试用例
- 添加边界条件测试

### 3. 非功能测试
- 从非功能需求提取性能、安全、兼容性测试需求

### 4. 测试数据
- 根据功能需求定义需要的测试数据

## 示例转换

### 输入（需求文档）
```
需求：用户登录功能
验收标准：
- 用户可以输入邮箱和密码
- 点击登录按钮成功登录
- 输入错误密码显示错误信息
```

### 输出（QA测试说明）
```
测试场景：
1. 正常登录流程
2. 密码错误情况
3. 邮箱不存在情况
4. 输入空值情况

测试用例：
TC001: 输入正确邮箱和密码，验证登录成功
TC002: 输入错误密码，验证显示错误提示
TC003: 输入不存在的邮箱，验证显示错误提示
...

性能测试：
- 登录响应时间 < 2秒
```
